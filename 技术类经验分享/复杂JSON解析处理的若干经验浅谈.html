💀背景简述 ​

项目xxx中对接云曜的IoT[1]消息事件，通过消息订阅的方式将设备事件上传给业务平台，而消息订阅的本质即配置一个接收端HTTP URL地址。

而其中复杂的地方在于设备的消息内容为字符串类型的泛型，根据消息体中的业务标识进行区分实际的消息类型；另外为了提高消息的处理效率，需要将消息消费设计为异步处理。

可能我把问题描述的有点绕，让人听得云里雾里。所以接下来把实际的协议报文摆出来，或许更加直观明了一些。

📌备注

关于云曜平台IoT消息事件配置相关的知识，可以参考前不久刚总结分享的wiki资料“Feign实战之对接云曜开放平台”。

协议结构 ​

根据上文的协议事例，整理出整个的协议结构如下，其中EventIdentifierEnum用于定义事件的类型，OtapProtocolMappingEnum用于建立事件类型和泛型实体类的映射关系，用于消息的反序列操作。

🏃设计目标 ​

由于消息体嵌套了多层泛型，所以结构体的定义和处理就会变得异常复杂。如果按照传统的处理方式，就需要定义很多个对象结构体，并且通过if...else...的硬代码方式进行判断处理。这样处理就会导致代码量变大的同时，还限制了逻辑的扩展性。

因为考虑到本次项目实际需要接入20+种类型的设备事件，所以有必要考虑设计一种通用可扩展的方式，大概罗列本次的设计目标如下：

  1. 消息事件的反序列化和业务处理逻辑完全解耦；
  2. 统一事件反序列化处理入口，独立抽象出数据解析逻辑；
  3. 支持消息异步多线程处理，提高事件的处理效率；
  4. 消息处理支持面向接口横向扩展，无需增加额外的对象反序列化逻辑；
  5. 事件类型和泛型类型有明确的映射关系，方便业务理解的同时能够具备良好的扩展性；

👀实现思路 ​

数据流程 ​

结合上文的消息协议特征和设计目标，梳理一下数据的整体流程。将云曜的消息在平台端处理拆分成消息入口，反序列化处理器，分发处理器和业务处理四部分。

其中消息的反序列化目前主流的方式有Gson、FastJson、Jackson[3]三种方式。关于这三种Json工具网上有铺天盖地的比对文章，所以本文将不做赘述，我可以分享一篇感觉不错的文档“JSON处理——FastJson、Jackson、Gson详细使用”。

关于消息异步消费能够想到的技术实现方式有：线程异步（继承Thread），Future异步，Spring的@Async异步等等，网络上有详细的几种技术方式优劣势比对的软文，自行查看吧。

而为了实现消息的业务分发，一种思路就是通过判断消息对象和接口入参对象方式确定具体的业务接口，或者通过枚举映射对象和接口的关系，但是这种方式扩展性会差一些；另外的一种思路是通过“观察者模式”设计模式写一套具备扩展性的消息处理逻辑。

但是在我学习了解异步消息处理过程中，发现了Spring的ApplicationEvent事件处理（我孤陋寡闻，之前确实没有使用过，很惭愧~）。在查阅了一些资料之后，发现ApplicationEvent如果加以巧用，能够完美解决上文提到的异步消费和业务分发需求，美滋滋。

总体实现思路 ​

我们将接收到事件字符串反序列化成POJO对象；通过ApplicationEventPublisher接口推送消息，

发布者并不需要考虑谁去监听，监听具体的实现内容是什么，发布者的工作只是发布事件而已。而消息的监听者也不需要知道消息来自于哪里，只要满足接口对象，就会被执行。而这一切全部由ApplicationEvent内部实现。

对于使用者来说，即“对象”决定业务处理。

"对象"决定业务 ​

📌题外话

这里的"对象"不是男女对象哦，是Java的Object对象。

一个对象即确认一种业务处理，因此对我们来说，现在只需要关心如何解析“对象”。

结合上文提到的消息结构体，如何实现MessageStr---》EventMessageDTO---》EventMessageListener这个生命周期的转换呢？

在这个过程中，我们需要解决对象的反序列化以及泛型对象实例化问题，关于反序列化会用到Jackson，而泛型实例化转换会用到Spring的ResolvableType。

🙇总体设计 ​

对象反序列化 ​

在MessageSubscriptionService消息处理类中通过Jackson的ObjectMapper工具进行消息字符串反序列化成POJO对象。

当然这里如果想要将数据反序列化的过程变得可扩展，可以将数据反序列化抽象成相应的接口，那么这样子就能支持扩展使用Gson、FastJson等其他Json工具进行处理。

事件对象映射 ​

为了将业务消息和具体的业务对象建立起关联关系，所以引入看一个MappingEnum枚举类，其中EventIdentifierEnum表示业务消息类型，bvClass表示消息体中基础属性数据泛型类型，tvClass表示消息体中时间戳数据泛型类型。

然后在消息处理Service中，通过枚举获取相应消息对象的泛型类，用于Json工具反序列化。

泛型对象实例化 ​

ApplicationEvent中通过ApplicationEventMulticaster控制器进行消息的分发，而对于泛型对象的匹配问题，则是通过ResolvableType resolveDefaultEventType(ApplicationEvent event)方法获取ResolvableType得到具体的泛型类型。

ResolvableType内部则是通过ResolvableTypeProvider接口获取，即对象通过实现ResolvableTypeProvider接口来实现消息分发中泛型对象的匹配问题。

/**
 * OTAP协议映射 [领域标识-业务标识-对象] 枚举。<br/>
 * 领域标识详见 {@link DomainIdentifierEnum}；事件标识 {@link EventIdentifierEnum} 。<br/>
 * 事件数据结构体详见 {@link EventMessage}，其中的<code>body<code/>数据结构体详见 {@link IotMessage}，{@link IotMessage#identifier}区分不同的事件类型。<br/>
 * 然而具体的设备事件数据最后通过{@link String}类型的<code>payload<code/>透传，具体结构体详见{@link WatchEventMessage}。<br/>
 * 看完{@link WatchEventMessage}结构体或许你就明白我为什么会定义这个枚举类了吧，其中的泛型<code>BV</code>对应<code>basicValue</code>，即{@link BV}接口标记类；
 * 泛型<code>TV</code>对应<code>timeValue</code>，即{@link TV}接口标记类，两者均为泛型动态结构体。<br/>
 * 这个事件结构体是不是超级吊啊，我真的是日了狗了。
 *
 * @author yuanzp
 * @date 2023/6/2 10:21
 * @digression Fucked the dog.
 */
@Getter
@AllArgsConstructor
public enum OtapProtocolMappingEnum {

    /*************
     * 设备状态
     ************/

    /**
     * 佩戴状态（设备->平台）
     */
    WEARING_STATE(EventIdentifierEnum.WEARING_STATUS, WearingStateBV.class),

    /**
     * 充电状态（设备->平台）
     */
    CHARGED_STATE(EventIdentifierEnum.CHARGING_STATE_REPORT, ChargedStateBV.class),

    /**
     * 定位信息（设备->平台）
     */
    LOCATION_INFO(EventIdentifierEnum.LOCATION_INFO_REPORTING, LocationInfoBV.class),

    /**
     * SOS报警（设备->平台）
     */
    SOS_ALARM(EventIdentifierEnum.SOS_ALARM, SosAlarmBV.class),

    //。。。此处省略若干代码。。。
    ;

    /**
     * 领域标识
     */
    @JsonValue
    private DomainIdentifierEnum domainIdentifierEnum;
    /**
     * 事件标识
     */
    @JsonValue
    private EventIdentifierEnum eventIdentifierEnum;

    /**
     * 基础数据对象类
     */
    private Class<? extends BV> bvClass;

    /**
     * 事件戳数据对象类
     */
    private Class<? extends TV> tvClass;

    OtapProtocolMappingEnum(EventIdentifierEnum eventIdentifierEnum, Class<? extends BV> bvClass, Class<? extends TV> tvClass) {
        this.eventIdentifierEnum = eventIdentifierEnum;
        this.bvClass = bvClass;
        this.tvClass = tvClass;
    }

    OtapProtocolMappingEnum(DomainIdentifierEnum domainIdentifierEnum, EventIdentifierEnum eventIdentifierEnum, Class<? extends BV> bvClass) {
        this.domainIdentifierEnum = domainIdentifierEnum;
        this.eventIdentifierEnum = eventIdentifierEnum;
        this.bvClass = bvClass;
    }

    OtapProtocolMappingEnum(EventIdentifierEnum eventIdentifierEnum, Class clazz) {
        this.eventIdentifierEnum = eventIdentifierEnum;
        if (BV.class.isAssignableFrom(clazz)) {
            this.bvClass = clazz;
        } else if (TV.class.isAssignableFrom(clazz)) {
            this.tvClass = clazz;
        }
    }

    public static OtapProtocolMappingEnum valueOfEventIdentifierEnum(EventIdentifierEnum eventIdentifierEnum) {
        for (OtapProtocolMappingEnum otapProtocolMappingEnum : OtapProtocolMappingEnum.values()) {
            if (otapProtocolMappingEnum.eventIdentifierEnum.equals(eventIdentifierEnum)) {
                return otapProtocolMappingEnum;
            }
        }
        return null;
    }

    public Class<? extends BV> getBvClass() {
        return getClass(bvClass);
    }

    public Class<? extends TV> getTvClass() {
        return getClass(tvClass);
    }

    private Class getClass(Class clazz) {
        return Optional.ofNullable(clazz).orElse(Object.class);
    }
}

/**
 * 事件对接，通过Spring {@link ApplicationListener}的实现，原理为<b>观察者模式</b>。<br/>
 * 如果需要新增对接一种iot手环协议事件，先在{@link OtapProtocolMappingEnum}中增加事件类型和实体DTO的映射关系用于实体转换（此转换为自动实现，尽量勿动此逻辑），
 * 接着实现消息监听器{@link EventListener}，增加必要的异步执行{@link Async}注解。<br/>
 * 例如参考“佩戴状态事件”{@link OtapProtocolMappingEnum#WEARING_STATE}，其事件监听器为{@link WearingStateListener}。
 * @param data
 * @return
 */
@PostMapping(value = "/event", produces = "application/json; charset=utf-8")
public BaseResult event(@RequestParam String data) {
	log.debug("IOT message:\n{}", data);
	//FIXME：本项目手环事件统一都是通过OTAP协议上报（即IOT协议），所以暂只处理该类型事件。
	List<IotMessage> body = Optional.ofNullable(data)
			.filter(StrUtil::isNotBlank)
			.map(str -> (EventMessage<IotMessage>) messageSubscriptionService.toPojo(str, EventMessage.class, IotMessage.class))
			.map(EventMessage::getBody)
			.orElse(null);
	if (CollUtil.isEmpty(body)) {
		log.warn("The event message body is empty.");
		return SUCCESS;
	}
	messageSubscriptionService.event(body);
	return SUCCESS;
}

/**
 * IOT事件处理
 * @param body
 */
public void event(List<IotMessage> body) {
	body.forEach(iotMessage -> {
		if (ObjectUtil.isNull(iotMessage)) {
			log.warn("The IOT message is null.");
			return;
		}
		EventIdentifierEnum eventIdentifierEnum = iotMessage.getIdentifier();
		OtapProtocolMappingEnum otapProtocolMappingEnum = OtapProtocolMappingEnum.valueOfEventIdentifierEnum(eventIdentifierEnum);
		if (ObjectUtil.isNull(eventIdentifierEnum) || ObjectUtil.isNull(otapProtocolMappingEnum)) {
			log.warn("IOT message identifier unsupported type.");
			return;
		}
		Object obj = Optional.ofNullable(iotMessage.getPayload()).filter(StrUtil::isNotBlank)
				.map(payload -> this.toPojo(payload, WatchEventMessage.class, otapProtocolMappingEnum.getBvClass(), otapProtocolMappingEnum.getTvClass()))
				.map(watchEventMessage -> BeanUtil.copyProperties(watchEventMessage, WatchEventMessageDTO.class))
				.map(watchEventMessageDTO -> {
					watchEventMessageDTO.setDeviceId(iotMessage.getDeviceId());
//                        watchEventMessageDTO.setUserId(iotMessage.getUserId());
					return watchEventMessageDTO;
				})
				.orElse(null);
		if (ObjectUtil.isNull(obj)) {
			log.warn("IOT message payload is blank.");
			return;
		}
		publishMessage(obj);
	});
}

/**
 * 对象转换
 * @param content 消息字符串
 * @param parametrized 对象类
 * @param parameterClasses 对象参数泛型类
 * @param <T>
 * @return
 */
public <T> T toPojo(String content, Class<T> parametrized, Class<?>... parameterClasses) {
	Assert.notBlank(content);
	Assert.notNull(parametrized);
	T o = null;
	try {
		if (ObjectUtil.isNull(parameterClasses)) {
			o = objectMapper.readValue(content, parametrized);
		} else {
			JavaType type = TypeFactory.defaultInstance().constructParametricType(parametrized, parameterClasses);
			o = objectMapper.readValue(content, type);
		}
	} catch (JsonProcessingException e) {
		log.error("Deserialization JSON object exception.",e);
	}
	return o;
}

/**
 * 消息推送
 * @param event
 */
private void publishMessage(Object event) {
	applicationEventPublisher.publishEvent(event);
}

/**

 * <code><pre>
 * {
 * 	"messageId": "639845999636672512",
 * 	"msgReceiveTime": 1684567174077,
 * 	"productCode": "1684399104246229",

 * @date 2023/5/24 11:31
 * @since 国家铁路局职工健康在线平台建设
 */
@Data
public class EventMessage<T> extends MessageSubscriptionParam implements ResolvableTypeProvider {
    /**
     * 消息体
     */
    private List<T> body;

    @Override
    public ResolvableType getResolvableType() {
        return Optional.ofNullable(this.getBody()).filter(CollUtil::isNotEmpty).map(CollUtil::getFirst).map(T::getClass)
                .map(clazz -> ResolvableType.forClassWithGenerics(this.getClass(), clazz)).orElse(null);
    }
}

 * 		"dateTime": "2021-01-28T02:00:00+08:00",
 * 		"UUID": "079f23cd-0988-459f-96f5-fa1c507dd07c"
 *    },
 * 	"payload": {
 * 		"basicvalue": {
 * 			//泛型，不同事件不同结构体
 *        },
 * 		"timevalue": {
 * 			//泛型，不同事件不同结构体
 *        }
 *    }
 * }
 * </pre></code>
 * @param <BV, TV> BV:基础属性;TV:时间戳数据
 * @author yuanzp
 * @date 2023/6/2 13:53
 * @since 国家铁路局职工健康在线建设
 */
@Data
public class WatchEventMessage<BV, TV> implements ResolvableTypeProvider {
    /**
     * 事件信息
     */
    private Basic basic;

    /**
     * 事件内容
     */
    private Payload<BV, TV> payload;

    @Override
    public ResolvableType getResolvableType() {
        return Optional.ofNullable(this).map(WatchEventMessage::getPayload).map(payload -> {
            ResolvableType bvResolvableType = Optional.ofNullable(payload.getBasicValue())
                    .map((Function<BV, ? extends Class<?>>) BV::getClass)
                    .map(ResolvableType::forClass).orElse(null);
            ResolvableType tvResolvableType = Optional.ofNullable(payload.getTimeValue()).filter(CollUtil::isNotEmpty)
                    .map(CollUtil::getFirst).map(TimeValue::getValue).map((Function<TV, ? extends Class<?>>) TV::getClass)
                    .map(ResolvableType::forClass).orElse(null);
            return ResolvableType.forClassWithGenerics(this.getClass(), bvResolvableType, tvResolvableType);
        }).orElse(null);
    }

    /**
     * 事件基础信息
     */
    @Data
    public static class Basic {
        /**
         * 设备第一个网卡的ipv4地址
         */
        private String ipV4Address;

        /**
         * 设备第一个网卡的ipv6地址
         */
        private String ipV6Address;

        /**
         * 设备第一个网卡的MAC地址
         */
        private String macAddress;

        /**
         * 时间
         */
        private String dateTime;

        /**
         * 上传事件唯一标识
         */
        @JsonProperty(value = "UUID")
        private String uuid;

    }
}

如何新增一种事件呢？ ​
