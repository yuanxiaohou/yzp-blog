H5Player组件使用浅析 ​

背景简述 ​

随着部分生产环境在推进硬件及软件国产化的工作，导致我们平台部分依赖于操作系统的功能无法使用。而本次项目上就遇到了录像回放功能无法正常使用的问题[1]。

在hicoo上经过一番搜索，发现公司恰好本身就已经有H5视频回放插件。但是这个工具怎么启用呢？兼容性怎么样？具体需要哪些参数数据？带着一堆的问号，开始了查找wiki资料和王工李工摇人的艰难历程。

而我整理这样一个文档的初衷，正是希望把我自己了解到的知识点以及测试通过的代码放给大家，或多或少能给后来人提供一点启发和经验。

📢 涉及组件版本信息 ​

h5player（H5视频播放器）：V2.0.1

vnsc（视频联网共享）：V1.9.103

dac(设备接入框架)：V1.10.102

sac（存储接入组件）：V1.4.104

vod（视频点播）：V5.9.103

（注：不提任何组件版本信息的参考文档基本就是在耍流氓。）

H5Player工具入门 ​

0、从hicoo上下载h5player工具[3]，进行解压；

1、将**\tool目录[4]下的\static文件夹，index-vue.html和server.js拷贝到\bin**目录中，并且顺带将index-vue.html改名为index.html。

2、在**\bin目录中打开cmd命令窗口，使node server.js**启动Demo服务[5]；

3、使用chrome80+[6]浏览器访问Demo页面，一张好看兮兮的页面就出来了。

页面是出来了，那么问题来了，里面涉及的那些URL地址该如何获取呢？因为上面“总体交互逻辑”章节已经提及对应ws url获取组件源，接下来就详细讲下这两个接口的请求报文，响应报文以及报文事例。

实时预览接口 ​

基本信息 ​

接口描述：

Web端获取实时预览连接信息

Path： /mls/service/rs/v1/preview/bs/getPreviewParam

Method： POST

请求参数 ​

Headers

参数名称参数值是否必须备注
Content-Typeapplication/json是JSON数据格式提交请求参数
token是接口校验令牌
properties
Content-Type:application/json
token:SElLIFd2eUF4b0hlU1JKYUFRcHY6YklCdUtqOWptdnc4YllxOVBUdWtVYVpLWGFCL3c5NU45WHhsS2FHK1JBYz06MTYzNDI2MDgxOTk4MQ==

Body

名称类型是否必须默认值备注
indexCodestring是监控点编号
netZoneCodeinteger否0网域编号（0:默认网域）
streamTypeinteger否1码流类型（0:主码流；1:子码流；2:第三码流；
254: 双码流搜索（优先返回主码流录像，
没有主码流录像时返回子码流录像）；
255:返回所有。未填或传入参数未定义默认为254）
protocolstring否rtsp传输协议（RTSP：”rtsp”； WebSocket：”ws”；
RTMP：”rtmp”； HLS：”hls” 默认为RTSP协议）
transmodeinteger否1协议类型（0: UDP；1: TCP 未填或传入参数未定义默认为TCP）
json
{
  "indexCode":"d92d0c4c9b8e435b90dc40ba69c663d2",
  "netZoneCode":0,
  "streamType":0,
  "protocol": "ws",
  "transmode":1
}

返回数据 ​

名称类型是否必须默认值备注
urlstring是实时预览URL地址
json
{
    "type": 0,
    "code": "0",
    "msg": "Success.",
    "data": {
        "url": "ws://10.19.189.172:7314/EUrl/V6YOnLi"

录像回放接口 ​

基本信息 ​

接口描述：

Web端获取录像回放连接信息

Path： /sac/sam/service/rs/v1/playback/bs/getPlaybackParam

Method： POST

请求参数 ​

Headers

参数名称参数值是否必须备注
Content-Typeapplication/json是JSON数据格式提交请求参数
token是接口校验令牌
properties
Content-Type:application/json
token:SElLIFd2eUF4b0hlU1JKYUFRcHY6YklCdUtqOWptdnc4YllxOVBUdWtVYVpLWGFCL3c5NU45WHhsS2FHK1JBYz06MTYzNDI2MDgxOTk4MQ==

Body

名称类型是否必须默认值备注
indexCodestring是监控点编号
recordStyleinteger是存储类型（0-中心存储；1-设备存储）
startTimestring是开始时间
endTimestring是结束时间
netZoneCodeinteger否0网域编号（0:默认网域）
protocolstring否rtsp传输协议（RTSP：”rtsp”； WebSocket：”ws”；
RTMP：”rtmp”； HLS：”hls” 默认为RTSP协议）
recordTypestring否录像类型（空表示查全部）
transmodeinteger否1协议类型（0: UDP；1: TCP ，未填或传入参数未定义默认为TCP）
streamTypeinteger否254码流类型（0:主码流；1:子码流；2:第三码流；
254: 双码流搜索（优先返回主码流录像，
没有主码流录像时返回子码流录像）；
255:返回所有。未填或传入参数未定义默认为254）
dataTypeinteger否0存储数据格式（0-视频：设备录制录像；
1-视频文件：用户上传录像；默认值为0）
lockTypeinteger否0锁定类型（0：查询全部；1：查询未锁定；
2：查询已锁定，默认值为 0）
json
{
	"indexCode": "d92d0c4c9b8e435b90dc40ba69c663d2",
	"recordStyle": 0,
	"startTime": "2021-10-13T09:20:00.000+08:00",
	"endTime": "2021-10-13T09:20:20.000+08:00",
	"netZoneCode": "0",
	"protocol": "ws",
	"recordType": "",
	"dataType": 0,
	"lockType": 0
}

注意事项 ​

1、回放报错'0x0183082e：取流失败' ​

F12控制台输出报错'0x0183082e：取流失败，详情根据错误码在运管后台进行查询'，运管上根据错误码查询得到的错误描述是'AES解密失败'。

这个错误的原因是进行回放操作未传入token校验令牌，本身给出的demo页面未考虑这块，而且未给出任何具体警示。

解决办法是修改index.html页面大约348行左右的playbackStart回放函数，临时测试传入token校验令牌，即可实现回放操作，以下js代码供参考。

javascript
/* 回放 */
playbackStart() {
  let { player, mode, urls, playback } = this,
	index = player.currentWindowIndex,
	
	playURL = urls.playback,
	{ startTime, endTime } = playback

	startTime += 'Z'
	endTime += 'Z'
	
  //警告：JS_Play函数务必传入token属性！！！
  player.JS_Play(playURL, { playURL, mode , token:TOKEN }, index, startTime, endTime).then(
	() => {
	  console.log('playbackStart success')
	  this.playback.rate = 1
	},
	e => { console.error(e) }
  )
}

2、使用wss协议进行预览回放，报错'0x12f910000' ​

问题的原因是自签名证书导致浏览器不信任websocket请求，导致该请求被拦截。

因为此次只是进行播放器工具的技术预研，发现wss协议不行也就没有纠结，之后采用ws的方式继续测试。

不过在wiki上查到此问题的解决办法，具体请参考wiki软文： wss取流失败。
