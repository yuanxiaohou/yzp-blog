浅谈一种事件聚类去重的实现思路 ​

✂️背景简介 ​

在机场围界项目中，根据前端设备上报的智能分析事件来判断围界是否存在人为入侵和翻越的风险。目前的方案是根据视频设备上报的区域入侵和穿越警戒面两类设备事件作为参考依据，判断是否存在防区入侵风险。

但是面临的问题是：设备事件存在大量的误报以及重复报警。比如:XX防区靠近马路，一旦有人/车进入离开检测区域，发生位移即会产生多个区域入侵事件；在一些大风大雨的恶劣天气，因为风吹草动的原因，设备端也会产生大量的误报和重复入侵报警事件。

对于重复报警问题，了解到已有的方案是判断两个报警之间时间间隔大小在一个阈值内及判定为重复事件。就着这个事件去重的方案，本文从另外的角度提出一些不太一样的解决思路和方法，供大家参考。

📌 说明

本文主要讨论事件去重的问题，暂不展开考虑有关于事件准确性及去误报相关问题。

📌场景分析 ​

现状 ​

以宁夏机场调研数据为例，围界入侵判断包含区域入侵、穿越警戒面两种类型的报警。对一周的数据进行了统计，以防区为报警源，涉及39个有效防区，共计报警15万+（153216），其中01、37、35、02、34防区累计报警9.5万+。

根据用户反馈，无效报警主要原因有刮风导致的草、树木，通过查看运行数据发现，报警较多的为濒临道路区域人车通行的影响，沟渠位置误报警。

以下附上现场一周数据统计详情，以及若干围界现场实景照片。

问题成因分析 ​

根据现有的数据以及以上现场围界实际场景，分析重复报警的成因主要有如下几点：

  • 现场处于施工（维修）阶段，施工人员的频繁活动导致产生大量重复报警；
  • 大风大雨等恶劣天气，引发设备产生大量的重复报警（本质上这是误报）；
  • 工作人员定期的巡逻和测试，诱发产生重复报警；
  • 设备由于未知原因产生大量重复报警（本质上这是误报）；

📂总体解决思路 ​

📌 说明

以围界入侵事件实现去重为例，其他事件按照实际数据结构参考处理即可。

按照人类判断两个事件是否重复的逻辑思维角度进行分析：接收到一个入侵事件，我们其实首先会根据时间，地点（事件源）想象刚刚是否发生过类似事件，然后根据事件图片中图形的轮廓是否在脑海中已经存在过进行联想，然后大脑自动帮我们关联到刚刚看到过的一个类似事件记忆，综上即判断该事件是否重复。

所以从中可以提取出几个人为判断依据的“数据原子”：事件类型、事件时间、发生地（经纬度或者相对位置）、事件图片。而对于计算机来说，判断两个事件是否重复即需要依赖于判定“数据原子”的相似度。

对于事件时间（一维数组），事件地点（二维数组）两组结构化数据的相似度计算方式为：设定阈值，然后计算差值和阈值做比较，进而判断两组数据的相似度。

而图片数据为三维组数数据结构，所以相似度的计算考虑使用 “感知哈希算法”[1]，它会给每一张图片生成一个“指纹”，然后比较两张图片的指纹，结果越接近就说明图片越相似。那么怎么比较两个指纹的相似度？且听下文分解。

📘相似度计算原理 ​

相似度计算公式是用来计算任何两个给定集合（数据或对象）之间的相似度的数学公式。它可以利用来做出各种技术方面的匹配和预测。相似度计算常常在机器学习、信息检素、数据挖据、图像识别等应用中使用。对于分类算法中，用其作为分类器叁数，以致于能够准确定位样本属性。

相似度计算常用于两个集合之间的相似度计算，如字符串的相似度，文本的相似度，图片的相似度等。常见的相似度计算公式有：欧氏距离[2]、曼哈顿距离、余弦距离[3]、皮尔逊系数[4]、杰卡德距离等。

它们均是把对象间的“距离”变为相似度，用值表示两者之间的差异，越大反映两个对象之间的差异越大，而越小则表示它们之间的差异越小。

时间相似度 ​

假设两个时间点分别为T1和T2，界限值（阈值）为θ，当然前提是阈值和差值同计量单位，相似度区间公式如下：

S(T1,T2)={0,∣T2−T1∣>θ1,∣T2−T1∣≤θS(T_1,T_2)= \begin{cases} 0,\quad|T_2-T_1|> \theta\\ 1,\quad|T_2-T_1|\leq \theta \end{cases} S(T​1​​,T​2​​)={​0,∣T​2​​−T​1​​∣>θ​1,∣T​2​​−T​1​​∣≤θ​​

上述数学公式解释：当两个时间点差值绝对值为0（即T1和T2同一时间点），即表示相似度为1；当两个时间差值绝对值大于界限值θ，即表示相似度为0。

具体的相似度计算公式为：

similarity(T1,T2)(∣T2−T1∣≤θ)=1−∣T2−T1∣θ{similarity(T_1,T_2)}_{(|T_2-T_1| \le \theta)}=1-\frac{|T_2-T_1|}{\theta} similarity(T​1​​,T​2​​)​(∣T​2​​−T​1​​∣≤θ)​​=1−​θ​​∣T​2​​−T​1​​∣​​

空间相似度 ​

假设待计算两点的位置信息为：L1(x1,y1)，L2(x2,y2)。采用欧氏距离公式进行计算，两点间距离计算公式为：

distance(L1,L2)=(x1−x2)2+(y1−y2)2distance(L_1,L_2)=\sqrt{(x_1-x_2)^2+{(y_1-y_2)^2}} distance(L​1​​,L​2​​)=√​(x​1​​−x​2​​)​2​​+(y​1​​−y​2​​)​2​​​​​

图片相似度 ​

比较两个图片的相似度在于得出图片指纹（数据集合），用这两个数据集合代入皮尔逊相关系数就可以得到。

那么怎么从图片得到图片指纹呢？

我们都知道任何图片在电子屏幕上显示都由光学三原色构成（红绿蓝），在计算机中我们对三原色用RGB来进行表示，比如RGB(102,255, 255) 表示的青色：

然后我们可以通过代码获取到一张图片的所有的像素点，然后把每个像素点进行拆分都可以得到相应的RGB数值，把这些RGB数值计数，那么就会得到：

text
(0,0,0) 出现次数10086次
(0,0,1) 出现次数25次
...
(255,255,255) 出现次数90次

把这些用次数收集起来，得到集合{10086,25,...,90}就是图片指纹。

但是如果我们对R,G,B都采用0-255的范围进行比较，那么产生的可能性就有：

2563=16777216256^{3}=16777216 256​3​​=16777216

所以为了简化计算，一般采用的是这样的方式：将0～255分成四个区：0～63为第0区，64～127为第1区，128～191为第2区，192～255为第3区。这意味着红绿蓝分别有4个区，总共可以构成64种组合（4的3次方）。

43=644^{3}=64 4​3​​=64

假设原色的值为：x，区块数为：n。

第一个区块的范围为：

[0,256n1−1]\left [ 0,\frac{256}{n} 1 -1\right ] [0,​n​​256​​1−1]

第二个区块的范围为：

[256n,256n2−1]\left [ \frac{256}{n},\frac{256}{n} 2 -1\right ] [​n​​256​​,​n​​256​​2−1]

那么第 i 次的区块范围为：

x∈[256n∗i,256n∗(i+1)−1]x\in \left [ \frac{256}{n} * i,\frac{256}{n} *(i+1) -1\right ] x∈[​n​​256​​∗i,​n​​256​​∗(i+1)−1]

此时这个 i 就是x应该放入区块的位置。

这里举个例子：假设原色的值为250那么带入上面的公式： 0～63为第0区，64～127为第1区，128～191为第2区，192～255为第3区，那么235属于第3区。

分别对两个图片都这样操作，然后得到这个数据集合，再通过上面说到的皮尔逊相关系数进行对比，就能计算出这两张图片的相似度了。

假设现有图片A和图片B，n表示区块数，Ai表示i区块出现次数（即指纹），A拔表示平均数（B参数同理可得），皮尔逊相关系数计算相似度的公式如下：

similarity(A,B)=∑i=1n(Ai−A‾)(Bi−B‾)∑i=1n(Ai−A‾)2∑i=1n(Bi−B‾)2similarity(A,B)=\frac{ \sum_{i=1}^{n}{(A_i-\overline{A})(B_i-\overline{B})} } { \sqrt{ \sum_{i=1}^{n}{(A_i-\overline{A})^2 \sum_{i=1}^{n}(B_i-\overline{B})^2} } } similarity(A,B)=​√​∑​i=1​n​​(A​i​​−​A​​​)​2​​∑​i=1​n​​(B​i​​−​B​​​)​2​​​​​​​∑​i=1​n​​(A​i​​−​A​​​)(B​i​​−​B​​​)​​

📌 说明

通常情况下通过相关系数取值范围判断变量的相关强度：

  • 0.8-1.0 极强相关
  • 0.6-0.8 强相关
  • 0.4-0.6 中等程度相关
  • 0.2-0.4 弱相关
  • 0.0-0.2 极弱相关或无相关

📚事件相似度 ​

由于各个项目实际情况，各个数据维度的之间的相似度并非等分，举个例子：比如A项目客户要求只根据时间相似度进行事件去重；或者B项目由于设备或者协议问题，拿不到图片，即无法通过图片相似度进行去重。综合考虑，结合以上的空间相似度，时间相似度，图片相似度的前提下，增加权重设计，权重越大对应的“数据原子”重要性就越大。

设定事件E1和E2的时间相似度为St，空间相似度为Ss，图片相似度为Si；时间相似度的权重为Wt，空间相似度的权重为Ws，图片相似度的权重为Wi。则相似度similarity计算公式如下：

similarity(E1,E2)=St×Wt+Ss×Ws+Si×WiWt+Ws+Wisimilarity(E_1,E_2)= \frac {St \times Wt + Ss \times Ws + Si \times Wi} {Wt+ Ws+ Wi} similarity(E​1​​,E​2​​)=​Wt+Ws+Wi​​St×Wt+Ss×Ws+Si×Wi​​

当然基于以上事件相似度计算公式，可以推导出多维的通用公式为：

similarity(E1,E2)=∑i∈nn(Si×Wi)∑i∈nnWisimilarity(E_1,E_2)= \frac{ \sum_{i\in n}^{n}{(S_i \times W_i)} } { \sum_{i\in n}^{n}W_i } similarity(E​1​​,E​2​​)=​∑​i∈n​n​​W​i​​​​∑​i∈n​n​​(S​i​​×W​i​​)​​

n为数据分析维度，Si为相似度，Wi为权重值。

📐历史数据分析 ​

上文中提到了相似度计算所需的阈值θ和权重W，接下来大概介绍下如何获取这两个参数的基准数值呢？

一方面是根据实际业务需求以及项目经验值设定一个基础标准，比如：时间阈值为5s，距离阈值为10m，阈值比为1：1：1。

另外一种思路是按照历史数据的离散程度，分析阈值以及权重对于实际数据的影响程度以及准确性。通俗来说就是根据项目上的历史数据，计算得到数据样本的平均差[5]或中位值[6]，然后人工校准的过程。

接下来以时间相似度数据分析为例，首先抓取一天n条事件数据，按照事件时间顺序排列，提取其中的时间T，计算出相邻数据时间差平均值T拔为：

T‾=∑ni=1(Tn−Tn−1)n\overline{T}= \frac{ \sum_{n}^{i=1}{(T_n - T_{n-1})} } { n } ​T​​​=​n​​∑​n​i=1​​(T​n​​−T​n−1​​)​​

所以如果将“时间相似度”提到的时间界限值（阈值）θ设置为平均值T拔，即大致得出结论：可将n条事件数据聚类成至多n/2条数据；且θ设置大于平均值T拔，聚类得到的数据会越少（误差越大），反之同理。

另外可以通过平均差公式，计算分析数据的离散程度，在一定程序上也可以作为阈值设置的参考，公式如下：

Md=∑ni=1∣(Tn−Tn−1)−T‾∣nM_d= \frac{ \sum_{n}^{i=1}{|{(T_n - T_{n-1})} - \overline{T}|} } { n } M​d​​=​n​​∑​n​i=1​​∣(T​n​​−T​n−1​​)−​T​​​∣​​

比如计算得到的平均差Md越大，表示此维度下数据的离散程度越高，即数据的相似度越低，这时候可以适当把θ设大一些，增加数据的聚类程度，反之同理。

当然以上推论都是以事件数据存在大量重复为前提条件，并且将推算得到的阈值数据导入实际样本数据，计算得出聚类数据之后，需要进行人工核验和校准。另外其他维度的数据分析手段雷同。
