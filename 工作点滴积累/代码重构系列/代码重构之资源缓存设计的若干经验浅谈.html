代码重构之资源缓存设计的若干经验浅谈 ​

前言 ​

这是本次OIM 2.3基线开发中“代码重构”系列[1]最后一篇“资源缓存设计的若干经验浅谈”。

通过分析缓存的加载，更新，调用几方面，提供一种相对比较统一的设计思路和实现方式。其实都谈不上什么设计，因为整个实现比较简单，如果有不周全的地方，还望指点。

现状分析 ​

现有缓存的实现主要是通过定义内存驻留对象ConcurrentHashMap进行数据缓存，然后所有的缓存加载以及使用全部在一个类中进行实现，有的会使用定时器@Scheduled进行定时刷新，总结出存在的问题大致如下：

  • 缓存存活在内存中，可见性较低，任务的执行灵活性也相对较差；
  • 破坏依赖倒置原则（面向接口编程），完全没有定义任何通用接口，这类通用资源的存取我还是极力推荐面向接口编程的；
  • 破坏迪米特法则（降低耦合），缓存代码和业务代码耦合度太高，不便于维护和变更；
  • 破坏开闭原则（对扩展开放，对修改关闭），这样的缓存方案不支持扩展依照某种条件的缓存，比如仅仅有某种类型的资源才须要缓存，这样的需求即会导致代码的修改甚至需要新写一个缓存；

如果对项目中缓存现状问题分析感兴趣的同事，可以参考“一种对象缓存的统一存取方式（壹：讲思路）”这篇wiki，这里面做过详细的问题分析。

总体设计思路 ​

核心的思路是将缓存的存取过程拆分为：缓存启动加载器（CacheLoadRunner）、缓存加载器（CacheLoader）、**缓存适配类（CacheRedisService）和缓存刷新定时器（CacheLoadTimer）**四大部分，最后通过Redis[2]实现分布式缓存的存取。

  • CacheLoadRunner：缓存启动加载器主要用于服务启动时候初始化加载缓存，保证应用启动完成之后即可快速使用缓存；
  • CacheLoader：缓存加载器负责实际缓存刷新加载的逻辑，一般的方式是先清除所有缓存，然后进行重新读取写入，相当于全量更新；
  • CacheRedisService：缓存适配类用于适配各种缓存具体的读写操作，写的操作一般比较固定，即初始化缓存的刷新set；而缓存的读则可以根据业务需求进行自定义；
  • CacheLoadTimer：缓存刷新定时器为了保证缓存与实际资源的一致性，通过xjob[3]实现定时任务的集成，也可通过页面进行手动触发，让定时任务具备一定的灵活性和可操作性；

📌 因为本次缓存的实现的主体是“资源”，所以下文的图例及代码主要笼统以ResourceCache为例，其中包括卡口、站点、车道和数据字典等等资源。

总体设计 ​

缓存加载刷新设计 ​

缓存初始化启动器ResourceCacheLoadRunner通过实现Spring的CommandLineRunner接口实现，可以通过@Order指定相应的执行顺序。在应用程序启动的同时，预加载所需的缓存资源。

在应用启动之后的生命周期内，则通过ResourceCacheTimer定时器进行定时刷新。定时器的内部实现依赖于xjob，在具体的定时任务方法上添加@XJobScheduled注解即可实现分布式定时任务。

缓存适配设计 ​

缓存的种类众多，但是基础的存取动作基本一致，即set、get等操作，所以通过定义抽象类AbstractRedisService实现基础的存取方法，而具体存储介质RedisTemplate则是通过具体缓存实现类进行注入，保证了一定的可扩展性。

其中资源缓存的实现更像是一种介于应用和存储介质之间的媒介。而且这个缓存类的定义是需要适配不同种类的缓存资源，以及具备一些业务较极强方法的扩展能力。

以卡口资源缓存类CrossCacheRedisService为例，在具有基础的根据卡口编号获取卡口对象的能力以外，根据业务需要，可能还需要具备根据卡口编号或者标签，获取对应的关联设备或者监控点方法。所以这个类一定是具有很强的适配性以及扩展性。

优化实现 ​

ResourceCacheLoader资源缓存加载器 ​

带权限的资源的加载主要通过ResourceRestServiceV2Client实现，所以将资源类读取统一实现，其中包括通用的while循环的分页加载。

java
/**
 * 资源缓存加载器<br>
 *
 * @author yuanzp
 * @date 2022/1/25 9:27
 */
@Slf4j
@Component
public class ResourceCacheLoader {
    @Resource
    private ResourceRestServiceV2Client resourceRestServiceV2Client;
    @Autowired
    private CrossCacheRedisService crossCacheRedisService;
    @Autowired
    private StationCacheRedisService stationCacheRedisService;
    @Autowired
    private UserServiceClient userServiceClient;
    @Autowired
    private LaneCacheRedisService laneCacheRedisService;
    @Autowired
    private RecordPlansCacheService recordPlansCacheService;
    @Autowired
    private RecordPlansRestServiceClient recordPlansRestServiceClient;

    private static final int DEFAULT_PAGE_SIZE = 500;

    public static final String RESOURCE_TYPE_STATION = "station";
    public static final String RESOURCE_TYPE_LANE = "LANE";

    /**
     * 卡口资源加载器
     */
    public void crossInfoCacheLoader() {
        crossCacheRedisService.deleteAll();
        resourceCacheLoader(SearchConstants.CROSS, (indexCode, resultMap) -> {
            crossCacheRedisService.set(indexCode, BeanUtil.mapToBean(resultMap, CrossInfoDTO.class, false));
        });
    }

    /**
     * 车道资源加载器
     */
    public void laneInfoCacheLoader() {
        laneCacheRedisService.deleteAll();
        resourceCacheLoader(RESOURCE_TYPE_LANE, (indexCode, resultMap) -> {
            LaneInfoDTO laneInfoDTO = BeanUtil.mapToBean(resultMap, LaneInfoDTO.class, false);
            laneCacheRedisService.set(laneInfoDTO.getCrossingIndexCode(), laneInfoDTO.getLaneDirection(), laneInfoDTO.getLaneNum(), laneInfoDTO);
        });
    }

    /**
     * 录像计划资源加载器
     */
    public void recordPlansCacheLoader() {
        recordPlansCacheService.deleteAll();
        recordPlansCacheLoader((indexCode, result) -> {
            //cameraIndexCode_type
            recordPlansCacheService.set(result.getCameraIndexCode(), result.getType(), result);
        });
    }

    private void recordPlansCacheLoader(CacheOperations<String, RecordPlansRespDTO> cacheOperations) {
        Date startDate = DateUtil.date();
        int i = 1;
        while (true) {
            BaseResult<BasePageDTO<RecordPlansRespDTO>> basePageBaseResult = recordPlansRestServiceClient.recordPlansPageSearch(i++, DEFAULT_PAGE_SIZE);
            BasePageDTO<RecordPlansRespDTO> basePage = Optional.ofNullable(basePageBaseResult).map(BaseResult::getData).orElse(null);
            List<RecordPlansRespDTO> recordPlansRespDTOS = Optional.ofNullable(basePage).map(BasePageDTO::getList).orElse(Collections.emptyList());
            if (CollUtil.isEmpty(recordPlansRespDTOS)) {
                break;
            }
            recordPlansRespDTOS.forEach(recordPlansRespDTO -> {
                cacheOperations.set(null, recordPlansRespDTO);
            });
            Integer totalPage = Optional.ofNullable(basePage).map(BasePageDTO::getTotalPage).orElse(0);
            if (totalPage < i) {
                break;
            }
        }
        long between = DateUtil.between(startDate, DateUtil.date(), DateUnit.MS);
        log.info("录像计划资源加载耗时： {} 毫秒.", between);
    }

    /**
     * 治超站资源加载器
     */
    public void stationInfoCacheLoader() {
        BaseResult<List<UserInfoDTO>> userResult
                = userServiceClient.findUserInfos(null, null, true);
        List<UserInfoDTO> userInfoDTOS = Optional.ofNullable(userResult).map(BaseResult::getData).orElse(null);
        if (CollectionUtils.isEmpty(userInfoDTOS)) {
            return;
        }
        stationCacheRedisService.deleteAll();
        userInfoDTOS.forEach(userInfoDTO -> {
            String userIndexCode = userInfoDTO.getUserIndexCode();
            stationInfoCacheLoader(userIndexCode);
        });
    }

    private void stationInfoCacheLoader(String userIndexCode) {
        resourceCacheLoader(userIndexCode, RESOURCE_TYPE_STATION, (indexCode, resultMap) -> {
            StationInfoDTO stationInfoDTO = Optional.ofNullable(stationCacheRedisService.get(indexCode))
                    .orElse(BeanUtil.mapToBean(resultMap, StationInfoDTO.class, false));
            Set<String> users = Optional.ofNullable(stationInfoDTO)
                    .map(StationInfoDTO::getUserIndexCodeSet).orElse(CollUtil.newHashSet(userIndexCode));
            users.add(userIndexCode);
            stationInfoDTO.setUserIndexCodeSet(users);
            stationCacheRedisService.set(indexCode, stationInfoDTO);
            stationCacheRedisService.set(stationInfoDTO.getCrossingIndexCode(), stationInfoDTO);
        });
    }

    /**
     * 通用资源加载函数
     * @param resourceType
     * @param cacheOperations
     */
    private void resourceCacheLoader(String resourceType, CacheOperations<String, Map<String, Object>> cacheOperations) {
        resourceCacheLoader(Constants.adminFlag, resourceType, cacheOperations);
    }

    /**
     * 通用资源加载函数
     * @param userId
     * @param resourceType
     * @param cacheOperations
     */
    private void resourceCacheLoader(String userId, String resourceType, CacheOperations<String, Map<String, Object>> cacheOperations) {
        Date startDate = DateUtil.date();
        int i = 1;
        while (true) {
            Param4ResourceV2 param4ResourceV2 = Param4ResourceV2.builder()
                    .pageNo(i++).pageSize(DEFAULT_PAGE_SIZE).resourceType(resourceType).userId(userId).build();
            BaseResult<BasePage<Map<String, Object>>> basePageBaseResult = resourceRestServiceV2Client.searchResourcesByAuth(param4ResourceV2);
            List<Map<String, Object>> resultMapList = Optional.ofNullable(basePageBaseResult).map(BaseResult::getData).map(BasePage::getList).orElse(null);
            if (CollUtil.isEmpty(resultMapList)) {
                break;
            }
            resultMapList.forEach(resultMap -> {
                cacheOperations.set(resultMap.get(SearchConstants.Resource_KEY).toString(), resultMap);
            });
        }
        long between = DateUtil.between(startDate, DateUtil.date(), DateUnit.MS);
        log.info("[{}]资源加载耗时： {} 毫秒.", resourceType, between);
    }
}

CacheRedisService资源缓存适配类 ​

因为资源缓存的适配类较多，所以挑选了卡口资源CrossCacheRedisService和数据字典DictionaryCacheRedisService两个比较具有代表性的类为例。

CrossCacheRedisService ​

java
/**
 * 卡口资源缓存
 *
 * @author yuanzp
 * @date 2022/1/25 9:05
 */
@Service
public class CrossCacheRedisService extends AbstractRedisService<String, CrossInfoDTO> {
    private static final String DEFAULT_KEY_PREFIX = "osurveillance:resourceCache:cross:";
    private static final String AT = "@";

    protected CrossCacheRedisService(@Qualifier("resourceCacheRedisTemplate") RedisTemplate<String, Object> redisTemplate) {
        super(redisTemplate);
    }

    @Override
    public String rebuildKey(String s) {
        if (StrUtil.startWith(s, DEFAULT_KEY_PREFIX)) {
            return s;
        }
        return StrUtil.concat(true, DEFAULT_KEY_PREFIX, s);
    }

    public String getCrossName(String indexCode) {
        return Optional.ofNullable(get(indexCode)).map(CrossInfoDTO::getName).orElse(StrUtil.EMPTY);
    }

    public List<String> getCrossTagList(String indexCode) {
        String tag = Optional.ofNullable(get(indexCode)).map(CrossInfoDTO::getTag).orElse(StrUtil.EMPTY);
        if (StrUtil.isBlank(tag)) {
            return CollUtil.newArrayList();
        }
        String[] split = StrUtil.split(tag, AT);
        return CollUtil.newArrayList(split);
    }

    public void deleteAll() {
        super.delete(super.keys(DEFAULT_KEY_PREFIX + Constants.ASTERISK));
    }
}

DictionaryCacheRedisService ​

java
/**
 * 数据字典资源缓存
 *
 * @author yuanzp
 * @date 2022/1/25 11:05
 */
@Service
public class DictionaryCacheRedisService extends AbstractRedisService<String, DataDictDTO> {
    private static final String DEFAULT_KEY_PREFIX = "osurveillance:dictionaryCache:";

    protected DictionaryCacheRedisService(@Qualifier("resourceCacheRedisTemplate") RedisTemplate<String, Object> redisTemplate) {
        super(redisTemplate);
    }

    @Override
    public String rebuildKey(String s) {
        if (StrUtil.startWith(s, DEFAULT_KEY_PREFIX)) {
            return s;
        }
        return StrUtil.concat(true, DEFAULT_KEY_PREFIX, s);
    }

    public String rebuildKey(String typeCode, String key) {
        return StrUtil.concat(true, typeCode, StrUtil.COLON, key);
    }

    public void set(String typeCode, String key, DataDictDTO value) {
        set(rebuildKey(typeCode, key), value);
    }

    public String getValue(DictTypeCodeEnum dictTypeCodeEnum, String key) {
        DataDictDTO dataDictDTO = get(rebuildKey(dictTypeCodeEnum.getQueryType(), key));
        return Optional.ofNullable(dataDictDTO).map(DataDictDTO::getDataValue).orElse(Constants.UNKNOW);
    }

    public String getName(DictTypeCodeEnum dictTypeCodeEnum, String key) {
        DataDictDTO dataDictDTO = get(rebuildKey(dictTypeCodeEnum.getQueryType(), key));
        return Optional.ofNullable(dataDictDTO).map(DataDictDTO::getName).orElse(Constants.UNKNOW);
    }

    /**
     * 判断数据字典键是否存在
     * @param dictTypeCodeEnum
     * @param key
     * @return
     */
    public Boolean hasKey(DictTypeCodeEnum dictTypeCodeEnum, String key) {
        return super.hasKey(rebuildKey(dictTypeCodeEnum.getQueryType(), key));
    }

    public void deleteAll() {
        super.delete(super.keys(DEFAULT_KEY_PREFIX + Constants.ASTERISK));
    }
}

AbstractRedisService ​

Redis抽象类主要提供基础的缓存存取服务，目前实现了RedisOperations、 ValueOperations、 ZSetOperations三类业务操作接口，但是其中具体的操作实现，需要自行通过注入的RedisTemplate进行实现。

        this.redisTemplate = redisTemplate;
    }
    
    /**
     * 重构key

     * @return

    /****Redis(要用方法的哥们儿，自己记得写实现！)****/

     * 设置缓存
     *
     * @param key
     * @param value

ResourceCacheLoadRunner资源缓存启动器 ​

通过实现CommandLineRunner接口，保证项目启动的同时进行缓存的加载，这里需要注意的是如果不进行try catch异常捕捉，会阻塞组件的启动，这个可以看各自的需求，是够要因为缓存的加载失败阻塞应用的启动。

java
/**
 * 资源缓存启动器
 *
 * @author yuanzp
 * @date 2022/1/25 11:45
 */
@Slf4j
@Component
@Order(value = 3)
public class ResourceCacheLoadRunner implements CommandLineRunner {
    @Autowired
    private ResourceCacheLoader resourceCacheLoader;
    @Autowired
    private DictionaryCacheLoader dictionaryCacheLoader;

    @Override
    public void run(String... args) {
        try {
            resourceCacheLoader.crossInfoCacheLoader();
            resourceCacheLoader.stationInfoCacheLoader();
            resourceCacheLoader.laneInfoCacheLoader();
            resourceCacheLoader.recordPlansCacheLoader();
            dictionaryCacheLoader.dictionaryCacheLoader();
        } catch (Exception e) {
            log.error("资源缓存初始化失败！",e);
        }
    }
}

ResourceCacheTimer资源缓存定时器 ​

将所有的资源缓存定时任务配置在ResourceCacheTimer定时器中，方便统一配置和管理。

其中任务cron[4]表达式通过配置文件进行配置（设置默认值），让定时任务的执行周期配置更加灵活。

java
/**
 * 资源缓存定时器
 *
 * @author yuanzp
 * @date 2022/1/25 11:40
 */
@Slf4j
@Component
public class ResourceCacheTimer extends BaseTimer {
    @Autowired
    private ResourceCacheLoader resourceCacheLoader;
    @Autowired
    private DictionaryCacheLoader dictionaryCacheLoader;

    @XJobScheduled(cron = "${hik.xjob.station-info-cache-timer.cron:0 0 */1 * * ? *}",
            taskId = "stationInfoCacheTimerOs",
            taskName = "站点资源缓存定时任务(osurveillance)", description = "定时(默认1小时)更新站点资源缓存")

        biu(() -> {
            resourceCacheLoader.stationInfoCacheLoader();
        });
    }

    @XJobScheduled(cron = "${hik.xjob.cross-info-cache-timer.cron:0 0 */1 * * ? *}",
            taskId = "crossInfoCacheTimerOs",

创新点及效果 ​

本案例分析总结了关于资源缓存方面的经验，设计实现了一种缓存初始化加载，更新，统一存取的方案，创新点及效果大致如下：

1、将缓存生命周期进行拆分解耦，在统一加载管理的前提下，也具备一定横向缓存类的扩展能力；

2、通过分布式缓存Redis进行存取，在必要时候提高了缓存的可见性和可操作性；

3、引入xjob分布式任务管理，让缓存的更新加载变得更加灵活可控；

4、遵守缓存加载的规则前提下，支持多个程序段做到缓存共享，提高了缓存的一致性；
