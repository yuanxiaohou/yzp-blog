软件产品国产化之PhantomJS适配 ​

背景简介 ​

TOCC产品[1]适配国产化开发的过程中，发现有组件中使用了PhantomJS用于生成Word报表里面的图表图片。因为以前部署服务器Linux是x86版本，毋庸置疑这在国产化arm版本的麒麟操作系统中将无法使用。

那么就需要自己编译或者找一个arm版的PhantomJS以满足项目要求。因为已有的国产化环境都在内网，且有网络限制，所以基本打消了自己编译的念头。在CSDN、gitee、GitHub上一通“翻箱倒柜”，最终被我找到了一个可用的版本，过程虽然略显坎坷，但结果还算喜人。

借着这次机会，我也大概重新认识学习了下这个工具，那么接下来就展开探索下PhantomJS吧。

PhantomJS浅析 ​

PhantomJS是什么？ ​

PhantomJS是一个自带JavaScript API的无头WebKit脚本。 它对各种web标准有快速和原生化的支持，包括DOM处理、CSS选择器、JSON、Canvas和SVG。

（题外话：PhantomJS 2.0以前的版本以花卉命名，特别有趣，感兴趣的可自行查阅。）

控制台 ​

有时你急需通过一小段Javascript代码来测试x或y是如何运作的， 通常你会打开Firebug中的Javascript控制台，Webkit Inspector， Opera Dragonfly或Chrome DevTools， 然后输入测试代码。

现在，新增了一个REPL（交互解释器：Read-Eval-Print Loop）后你可以从PhantomJS中执行相同的操作了。

使用场景 ​

✔️网站无头测试：使用诸如Jasmine、QUnit、Mocha、Capybara、WebDriver等框架进行功能测试。

✔️网页截屏：以编程方式捕获web内容，包括SVG和Canvas。 使用缩略图预览创建网站截图。

✔️页面自动化：使用标准DOM API，或者类似像jQuery库访问和操作网页。

✔️网络监控：可将监视页面导出为标准的HAR文件。使用YSlow和Jenkins自动化性能分析。

问题1：执行phantomjs报错“无法执行二进制文件: 可执行文件格式错误” ​

解压完PhantomJS之后，通过命令行./phantomjs -v直接报错“无法执行二进制文件: 可执行文件格式错误”，大致如下：

-bash: ./bin/phantomjs：无法执行二进制文件: 可执行文件格式错误

错误原因

PhantomJS二进制执行文件非arm环境编译生成，导致无法执行。

解决办法

网上下载arm版PhantomJS或者使用本文附件提供的“phantomjs_2.1.1_arm64.zip”，然后测试验证。

问题2：执行phantomjs报错“ error while loading shared libraries:XXX” ​

解压完PhantomJS之后，通过命令行./phantomjs -v直接报错“./phantomjs: error while loading shared libraries: libssl.so.10: cannot open shared object file: No such file or directory”。

错误原因

PhantomJS二进制执行文件编译成果物在当前环境缺少依赖库。

解决办法

段错误

错误原因

echarts-convert内部引用的echarts.js有问题，应该是底层的渲染引擎不兼容。

解决办法

可以使用echarts_convert_arm64.zip包里面的echarts.min.js，或者去ECharts官网定制导出一份js（参考：https://blog.csdn.net/muriyue6/article/details/105773711），然后进行验证测试。

结束语 ​
