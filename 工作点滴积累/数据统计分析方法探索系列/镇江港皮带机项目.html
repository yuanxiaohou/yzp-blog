镇江港皮带机项目 ​

序言 ​

从学术角度来看，数据统计分析是对数据进行收集、整理、描述和定量分析的一个过程，以此来探索数据背后的规律和趋势；从工作角度来看，应用统计、分析学知识，可以更为科学的计算和度量数据对业务的价值。

日常的项目中，我们会面对各种各样，五花八门的基础数据，比如视频数据，图片数据，雷达探测数据以及各种各样的融合事件等等，而如何从这些数据中探索可能存在的业务场景以及能够解决的业务问题呢？值得让人思考，这个过程中逃脱不了对数据进行统计、分析和研判。

因此，为了更好的探索数据价值和解决项目问题，掌握基础的统计学知识显得非常有必要。而我作为一个“门外汉”，想要通过日常项目的点滴积累和记录，来慢慢探索这门学科以及留下点什么。

项目背景 ​

皮带机项目通过接入现场的相机、骨传导设备、听诊光纤设备、探伤设备、流量监测激光雷达设备之后，对相关的数据进行智能分析和过滤，提供皮带跑偏检测、落料堆积检测、电机异响检测、滚筒异响检测、皮带损伤检测、皮带堵料检测能力，以此研判皮带线运输中的异常运行状态，为值班人员和设备技术运维人员提供数据支撑。

本文主要分享皮带堵料检测业务场景的研判过程，皮带机在生产运行中因为大块物料、物料粘稠等原因在下料漏斗和导料槽之间可能出现物料大量堆积，堵塞下料溜槽的情况，发现不及时将影响生产。

目前平台通过雷达设备，采集皮带机上的进出（以2A和3A皮带作业线为例）流量做差值进行分析，判断是否发生物料堵塞风险。但是目前堵料报警效果并不理想，所以拟通过接入作业线上的皮带秤[1]数据和雷达原始体积数据，辅助分析原始数据的准确性和二次研判堵料隐患。

物理布设 ​

本次主要分析2A皮带作业线和3A皮带作业线之间的设备以及数据。

其中设备包括：皮带秤，雷达主机盒子（接入两路前端雷达设备），IPC监控；接入的数据包括：皮带秤作业数据（作业任务，作业开始时间，货物名称，传授物料累加重量t），雷达体积数据（单位时间区间内传送物料的体积m³）；另外还有一些基础物理参数：比如雷达和皮带秤之间的距离，皮带秤的运行速度和两台雷达之间的数据时差。

数据基础 ​

基础数据包括两部分：第三方SQL Server数据库的皮带秤数据，包括TbClientDataWorkLine皮带秤作业线数据表和TbClientDataYB皮带秤实时仪表数据表；前端雷达主机接入数据，包括堵漏报警和实时体积等数据。

皮带秤作业线数据表

皮带秤作业线数据表TbClientDataWorkLine存储皮带秤采集的作业信息，包括作业线号、货主、货物名称和皮带秤重量等属性。逻辑上数据不会增长，通过定时（频率为5秒）刷新的方式更新数据（备注：TbClientDataYB表同步逻辑基本同理）。

根据作业线信息定时刷新皮带称的重量信息Weigh（自开始作业时间的质量累加值，单位是t）和数据时间UpdateTime。

皮带秤实时数据表

皮带秤实时数据表TbClientDataYB存储皮带秤采集的运行速度和皮带秤重量等属性。其中重量YbWeight为累计值，速度YbSpeed为实时数据。

雷达设备

雷达设备包括前端探测雷达终端和雷达主机（负责基础数据接入，存储，读研研判和转发等等）。

数据通过DAC[2]驱动方式（天河电子私有协议雷达驱动drv_ias_radardevice_galaxy_net）接入，数据包括：物料卡堵事件，物料瞬时体积数据，物料流量数据等。

分析目标 ​

本次数据分析，主要为了验证两个问题：其一是用皮带秤的称重数据来核对平台入库的雷达体积数据是否准确（通过分析密度趋势，原理参考“质量和体积数据分析”章节）；其二是通过两个雷达数据的变化趋势互相验证数据准确性以及分析堵料风险（原理参考“雷达数据互证分析”章节）。

准备工作 ​

💭 备注

  • 皮带线作业安全监管应用：bmwsm_1.0.0001；
  • 数据库连接工具：DBeaver Enterprise 7.1.0；

皮带秤数据抽取 ​

平台实现将三方SQL Server皮带称数据实时同步至PostgreSQL数据库（bmwsm组件中实现该逻辑），通过定时任务的方式实时抽取TbClientDataWorkLine和TbClientDataYB（对应到PG数据库的pdc_client_data_work_line和pdc_client_data_yb）。

除此以外大概说下表数据的抽取原理：即定时将A库的表数据查询出来直接插入B库的表中，其中用到了INSERT语句以及ON CONFLICT语法，防止数据的重复插入，SQL语法如下：

plsql
insert into 
	pdc_client_data_yb (uuid, scale_name, update_time, yb_weight, yb_speed, yb_flow)
values 
	(?,?,?,?,?,?) ,(?,?,?,?,?,?) 
on conflict (uuid) do nothing;

雷达数据接入 ​

从ESC订阅消费雷达的瞬时体积数据（雷达其他数据类型接入同理），事件类型eventType为3182596，实时（秒级）将消息数据存储到t_material_instant_volume表中。

内部表数据介绍 ​

本次数据分析包括两类数据（皮带秤的称重数据和雷达的瞬时体积数据），三张表（皮带秤作业线数据表，皮带秤实时数据表，雷达体积数据表）。接下来大概介绍一下表结构以及关联关系含义。

皮带秤作业线数据表（pdc_client_data_work_line）

皮带秤实时数据表（pdc_client_data_yb）

雷达体积数据表（t_material_instant_volume）

表结构关系 ​

pdc_client_data_work_line表和pdc_client_data_yb表可以通过皮带秤编号scale_name和数据时间update_time进行匹配关联。而皮带秤的重量数据和t_material_instant_volume表的体积数据通过时间和作业线ID进行关联。

数据分析思路 ​

皮带秤数据梳理 ​

皮带秤的重量数据是一个累加值，我们需要将时刻的累加值转换成为单位时间区间内的重量值，使用”逐差法“进行计算。

时间区间以分钟为例，实现方式即获取整点分钟数据，然后前后分钟的重量数据做差，约定重量差值为时间右区间时刻点，统计SQL语句和结果如下：

sql
select
	t1.t,
	t1.total_weight - t2.total_weight as weight
from
	(
	select
		to_char(update_time, 'YYYY-MM-DD HH24:MI') as t, count(*) as total_count, sum(weight)/ count(*) as total_weight
	from
		pdc_client_data_work_line
	where
		production_line = '202404050001'
	group by
		t) t1,
	(
	select
		to_char(update_time + interval '1 minutes', 'YYYY-MM-DD HH24:MI') as t, count(*) as total_count, sum(weight)/ count(*) as total_weight
	from
		pdc_client_data_work_line
	where
		production_line = '202404050001'
	group by
		t) t2
where
	t1.t = t2.t
order by
	t asc;

雷达数据梳理 ​

雷达的原始数据为时刻区间的体积值，我们只需要按照单位时间（以分钟为例）区间进行聚类加和即可，统计SQL语句和结果如下：

sql
select
	to_char(alarm_time, 'YYYY-MM-DD HH24:MI') as t,
	count(*) as total_count,
	sum(volume) as total_volume
from
	t_material_instant_volume
where
	alarm_time between '2024-04-05 07:47:23' and '2024-04-06 15:34:07'
	and radar_index_code = 'd07e114c4b8a402fbfce7ddca6260a7c'
group by
	t
order by
	t asc;

质量和体积数据分析 ​

假定皮带线运行稳定且原始数据准确，平台入库逻辑正常，无漏数据等异常，则一趟作业任务（传送同一类型物料）计算分析得到的密度因应该基本为一条恒定的直线。

ρ=mV\rho=\frac{m}{V} ρ=​V​​m​​

参考”物理布设“章节的布设图，可以发现2A作业线上的皮带秤和雷达设备的物理位置存在距离差，因此首先在数据上需要消除时差，即需要根据距离和运行速度计算出数据时差。

v=stv=\frac{s}{t} v=​t​​s​​

皮带机的运行速度可以从pdc_client_data_yb表获取，本次获取速度值只能通过统计得到一个均值（2A皮带线的运行均速为2.5米/秒），SQL语句如下；

sql
select
	speed as "2A皮带线速度",
	count(*) as  "速度值数量"
from
	(
	select
		round(yb_speed, 1) as speed
	from
		pdc_client_data_yb) pcdy
where
	speed > 0
group by
	speed
having
	count(*) > 10
order by
	speed desc;

根据物理距离和速度即可计算得到时差为24秒（t=60米÷2.5米/秒），质量和体积统计数据消除时差之后即可通过时间（本文以分钟为例）为关联条件进行关联查询并计算密度。

雷达数据互证分析 ​

假定皮带线运行稳定且原始数据准确，平台入库逻辑正常，无漏数据等异常，则2A雷达和3A雷达数据消除时差之后，按照单位时间（以分钟为例）得到的统计值曲线的变化趋势基本保持一致，即单位时间内的体积累加和差值基本恒定。

2A雷达和3A雷达数据的时差计算原理是：根据物理空间距离计算皮带线上的传送时差，原理同上文提到的时差计算公式6秒（t=15米÷2.5米/秒）；另外根据现场实际观察，两条皮带线之间物料通过导料漏斗中转的时间为1秒。结合两个计算得到的时间数据，最终得到的数值为7秒，

然后结合”雷达数据梳理“章节得到2A雷达和3A雷达的基础体积统计数据，消除时差之后即可根据时间为关联条件，关联查询得到体积的变化曲线。

分析结果 ​

通过数据库分析工具执行SQL，查询对应时间区间内的统计数据，然后导出到Excel中转换为折线图表即可查看对应的数据变化趋势图。

密度数据分析 ​

通过关联质量和体积数据即可计算得到物体的密度。

sql
select
	t1.t as "时间（分钟）",
	t1.weight as "质量（t）",
	t2.total_volume as "体积（m³）",
	t1.weight / t2.total_volume  as "密度（t/m³）"
from
	(
	select
		t1.t, t1.total_weight - t2.total_weight as weight
	from
		(
		select
			to_char(update_time, 'YYYY-MM-DD HH24:MI') as t, count(*) as total_count, sum(weight)/ count(*) as total_weight
		from
			pdc_client_data_work_line
		where
			production_line = '202404050001'
		group by
			t) t1, (
		select
			to_char(update_time + interval '1 minutes', 'YYYY-MM-DD HH24:MI') as t, count(*) as total_count, sum(weight)/ count(*) as total_weight
		from
			pdc_client_data_work_line
		where
			production_line = '202404050001'
		group by
			t) t2
	where
		t1.t = t2.t) t1,-- 皮带秤分钟重量统计数据
	(
	select
		to_char(alarm_time - interval '24 seconds', 'YYYY-MM-DD HH24:MI') as t, -- 按照皮带秤和雷达物理空间距离，运行速度计算得到的时差
		count(*) as total_count, 
		sum(volume) as total_volume
	from
		t_material_instant_volume
	where
		alarm_time between '2024-04-05 07:48' and '2024-04-06 15:34'
		and radar_index_code = 'd07e114c4b8a402fbfce7ddca6260a7c'
	group by
		t) t2--雷达分钟体积数据
where
	t1.t = t2.t
	and t1.weight>0
	and t2.total_volume>0
order by
	t1.t desc;

通过图表可以很直观的看到密度变化曲线以及质量和体积的变化曲线，可以发现密度基本呈直线状，即证明距离差、时差以及基础数据基本准确。

其中密度曲线中的一些”离散凸起“即可以初步判定为质量或者体积的异常点，即可能存在卡堵风险。

雷达数据比对分析 ​

通过关联2A和3A雷达点位的体积数据即可计算得到比对分析结果。

sql
select
	t1.t as "时间（分钟）",
	round(t1.total_volume::numeric , 2) as "2A体积（m³）",
	round(t2.total_volume::numeric , 2) as "3A体积（m³）",
	round(ABS(t1.total_volume -t2.total_volume)::numeric , 2) as "差值（绝对值）"
from
	(
	select
		to_char(alarm_time, 'YYYY-MM-DD HH24:MI') as t, count(*) as total_count, sum(volume) as total_volume
	from
		t_material_instant_volume
	where
		alarm_time between '2024-04-05 07:47:23' and '2024-04-06 15:34:07'
		and radar_index_code = 'd07e114c4b8a402fbfce7ddca6260a7c'
	group by
		t) t1,
	(
	select
		to_char(alarm_time - interval '7 seconds', 'YYYY-MM-DD HH24:MI') as t, count(*) as total_count, sum(volume) as total_volume
	from
		t_material_instant_volume
	where
		alarm_time between '2024-04-05 07:47:23' and '2024-04-06 15:34:07'
		and radar_index_code = '4ff22705a33e43fc8417c3f6989ed003'
	group by
		t) t2
where
	t1.t = t2.t
order by
	t1.t desc;

两个点位的体积变化曲线基本拟合，变化趋势保持一致，即”拟合度“较高，可证明距离差、时差以及基础数据基本准确。

假定体积差值超过一定的阈值（该阈值需要根据历史报警数据进行分析和评估），即判定可能存在卡堵风险。

注意事项 ​

  • 数据分析中涉及的物理参数，比如雷达和皮带秤之间的距离，皮带秤的运行速度和两台雷达之间的数据时差，这些需要根据实际分析场景进行合理计算和调整；
  • 皮带秤的质量数据和雷达的基础数据在计算过程中，比如使用分钟为时间区间统计，需要剔除掉首尾时刻的数据以消除误差，原因首尾数据通过”逐差法“或者”累加法“都存在不完整性；
  • 根据皮带秤的质量数据和雷达的体积数据计算密度分布曲线，需要保证是同一作业线趟次的数据，即pdc_client_data_work_line表中production_line为检索条件，对应时间区间内的t_material_instant_volume表雷达数据；
  • 雷达数据比对分析时候需要核验上报的雷达数据是否已经经过时差校对，判断依据可以是把两个雷达秒级统计数据的曲线拉出来，反向推测数据时差；

结束语 ​

本文通过分析项目中皮带秤的体积数据和雷达设备的体积数据，来计算研判数据的离散性和拟合度，在一定程度上实现了数据准确性互证和分析堵料风险。
