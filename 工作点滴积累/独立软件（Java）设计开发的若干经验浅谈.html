独立软件（Java）设计开发的若干经验浅谈 ​

背景简介 ​

业务介绍 ​

独立软件的开发需求来源于机场闸机项目，项目中的闸机硬件主要用于机场自助安检或离港系统等业务场景，其中工控机程序（即“独立软件”）主要用于统一对接控制闸机上的外设硬件：身份证读卡器，护照阅读器，屏显等等，另外还需要负责对接上级应用系统，起到一个“承上启下”的作用。

具体的业务流程及示意图可参考“RS-232串口通信实战开发之项目实战篇”中的“业务流程介绍”章节。

📌 备注

下文中的“工控机程序”、“程序”等名词同“独立软件”含义。

基础环境 ​

在开始分享之前，首先介绍下独立软件的基础运行环境，脚手架版本，JDK版本等信息，本次独立软件通过jc脚手架搭建，使用Java语言开发。

  • Windows 10专业版

  • aries-jc-dependencies：3.1.14.RELEASE

  • spring-boot-starter-parent：2.6.15

  • JDK：1.8.0_181-b13（ 64-Bit Server VM）

需求&设计 ​

服务自启动 ​

在实际应用场景中，服务器会存在频繁断电重启的情况，另外一些依赖的外部业务服务可能处于不可用的状态。

以本次对接经验为例，硬件闸机在工厂组装和调试的过程中，因为接线问题或者电压负载过高等问题，一天中被断电了七八次，另外在工厂中，网络上也无法对接到应用平台（悦泰）。

结合以上两点情况，服务自启动的设计即包含两个层面：一方面是部署在服务器上的独立软件需要支持服务器断电重启之后，能够自行启动；另外一方面是软件设计时候需要考虑到依赖服务不可用的情况，增加必要的异常情况容错设计，或者增加可配置开关，允许自行控制连接与否，保证不影响主程序的启动。

启动完成提醒 ​

一个独立运行在服务器中（或PC）的程序，日常运维过程中，比如程序重启，如何直观的感知到程序是否完全启动呢？

目前的手段一般是通过查看日志的方式进行判断。但是在实际操作过程中，这个多少有点不方便和不直观，由于闸机上本身配备了喇叭设备，所以考虑在工控机程序完全启动后，播放一段语音提示，让用户可以直观的感知到软件启动就绪，闸机也已经处于可用状态。

比如你站在两台冷冰冰的闸机前，两者同时断电重启：一台闷声不响，冷冰冰的呆在那儿，好像有点心事；另外一台在重启完成之后，播报了一段小姐姐甜美的“欢迎使用XXX，祝你生活愉快”之类的语音。这样子你应该能够体会到这个设计的必要性了吧。

当然也可以根据实际的设备条件情况，通过其他提示音或者灯光提示。另外声光提醒需要增加必要的开关配置，用户能够自主决定要不要在启动完成后进行提示。

可视化初始化安装 ​

一个独立软件在初始化安装时候，或多或少会需要一些配置信息。本案例为例，即需要配置读卡器，ARM权限板，屏显设备，应用平台地址等等信息。

比如研发兄弟甩给你这么一个配置文件，然后配合一个不是很详细的部署手册。暂且抛开配置的便捷程度不谈，我自己在操作的过程中都觉得需要谨小慎微，看着这么多密密麻麻的配置脑袋都大了，但凡在这个文件里面哪里多敲了一个字符，结果都需要付出“惨重代价”。

所以我想到了可视化初始化安装的设计：程序初始化安装完成之后，首次访问软件时候，页面可视化进行配置，配置完成之后重启程序即可进行正常的后续操作使用。

需要注意的是，以上流程中保存的配置信息并不会直接写入application.properties中，而是使用了spring.profiles.active特性（可参考“SpringBoot配置文件中spring.profiles.active配置详解”），保存到对应环境的配置文件中。此设计可以防止原始配置文件被串改，还可以支持不同环境的切换测试。

设备自检 ​

程序完成初始化安装配置之后，理论上即可进行正常的使用。但是由于是一个控制类程序，本身没有查询页面，也没有办法独立进行业务流程（比如自助安检的流程），但是设备在调试自测阶段又需要具备自检能力。工控机程序跑在设备上面，当批量出货的时候，即需要设计开发一种高效便捷的设备自检功能。

为了方便设备的自测，考虑设计一个设备自测的页面，通过简单的页面操作即可完成设备功能的独立测试。除此之外，为了提高自检的效率，还可以设计开发“一键自检”，通过程序轮询测试一遍外设设备。

集成&实现 ​

独立运行 ​

运行的独立性的前提主要包含如下两大方面：即不依赖于平台、运管、MQ、数据库、Consul等等远程服务依赖；另外独立的服务注册，管理，启动。主要包括的改造如下，当然实际项目实际分析。

  • start模块中，bootstrap.properties配置文件的aries.jc.configure.enabled设置为false，并且移除config.properties，installation.properties，startupConfig.xml，cas-client.properties文件。
  • business模块中，移除CAS，PostgreSQL，MQ等依赖。
  • 需要自行安装JDK安装，本案例使用的是JDK安装包（jdk1.8-windows-x64.zip，从附件中获取），直接通过双击安装即可，就不展开介绍了。
  • __service启动脚本改造，统一使用WinSW服务方式管理程序服务，具体方式可参考“WinSW服务自启动”章节。
  • assembly打包配置中根据实际打包包含关系修改。

WinSW服务自启动 ​

Windows下服务的注册以及自启动通过Windows Service Wrapper（简称“WinSW”）实现。WinSW是一个可执行二进制文件，可用于包装和管理作为Windows服务的自定义进程。

📌 备注

WinSW可以在安装了.NET Framework 4.6.1 或更高版本的Windows 平台上运行。对于没有 .NET Framework 的系统，该项目提供基于 .NET 7 的本机 64 位和 32 位可执行文件。

下文主要以官网事例为参考讲解，实际项目事例直接从附件中获取。

安装包下载

最新版本和预发布的 WinSW二进制文件可在GitHub Releases上找到。

然后根据.Net Framework版本下载即可，在地址栏上输入“C:\Windows\Microsoft.NET\Framework”，可以看到.NET Framework版本。如下图中所示即最高版本为4.0，即对应WinSW.NET4.exe。

服务配置

下载安装包后，您可以将 WinSW.exe 重命名为实际目标服务名称，例如 MyService.exe 。接下来创建服务同名的XML文件，例如MyService.xml文件进行服务配置。

参考官网的XML事例 xml-config-file.md：

xml
<service>
  <id>jenkins</id>
  <name>Jenkins</name>
  <description>This service runs Jenkins continuous integration system.</description>
  <env name="JENKINS_HOME" value="%BASE%"/>
  <executable>java</executable>
  <arguments>-Xrs -Xmx256m -jar "%BASE%\jenkins.jar" --httpPort=8080</arguments>
  <log mode="roll"></log>
</service>

相关参数说明如下：

  • id：安装windows服务后的服务ID，必须是唯一的。
  • name：服务名称，也必须是唯一的（一般和id一致即可）。
  • description：服务说明，可以使用中文，可做备注使用。
  • env：环境变量配置，内部默认带有环境变量BASE，该变量指向WinSW.exe所在目录。
  • executable：执行的命令，比如启动springboot应用的命令java。
  • arguments：命令执行参数，比如 包路径，类路径等。
  • log：日志记录，支持如下几种模式：append (default), reset (clear log), ignore, roll (move to \*.old)。

除此以外，还有几个常用的配置参数：

  • startmode：Windows启动服务模式：Automatic（自动），Manual（手动）。
  • logpath：日志文件路径。

启动完成提醒 ​

实现Spring CommandLineRunner来实现程序启动后的自动执行逻辑。可以通过@Order注解，指定加载顺序，默认值为Integer.MAX_VALUE。

java
/**
 * 程序自启动欢迎语提示，方便感知闸机程序的启动状态
 *
 * @author yuanzp
 * @date 2022/12/11 19:21
 * @digression 西宁曹家堡机场三期扩建工程信息弱电工程
 */
@ConditionalOnProperty(name = "welcome-voice-tips.enable", havingValue = "true")
@Configuration
@Slf4j
@Order()
public class WelcomeVoiceTips implements CommandLineRunner {
    @Autowired
    private SoundMaterialService soundMaterialService;

    @Override
    public void run(String... args) {
        try {
            soundMaterialService.play(SoundMaterialEnum.WELCOME_VOICE_TIPS);
            log.info("Welcome voice tips played successful.");
        } catch (Exception e) {
            log.error("Welcome voice tips played error.", e);
        }
    }
}

声音播放逻辑，通过Clip对象实现，目前只支持播放wav类型的音乐文件，代码逻辑如下：

java
/**
 * 声音服务<br/>
 * <i>注意：目前只测试通过wav类型音乐文件</i>
 *
 * @author yuanzp
 * @date 2023/11/13 17:58
 * @digression done is better than perfect.
 */
public class Sounder {
    private Logger LOG = LoggerFactory.getLogger(Sounder.class);
    private Map<String, Clip> clipMap = new HashMap<>();

    public Sounder(List<Pair<String, File>> audioFiles) {
        audioFiles.forEach(audioFilePair -> {
            add(audioFilePair);
        });
    }

    /**
     * 加载音频
     * @param audioFilePair
     */
    public void add(Pair<String, File> audioFilePair) {
        String audioCode = audioFilePair.getKey();
        File audioFile = audioFilePair.getValue();
        try {
            AudioInputStream audioInputStream = AudioSystem.getAudioInputStream(audioFile);
            Clip clip = AudioSystem.getClip();
            clip.open(audioInputStream);
            clipMap.put(audioCode, clip);
        } catch (Exception e) {
            LOG.error("Sounder init error", e);
        }
    }

    /**
     * 播放音乐（覆盖式）
     * @param audioCode
     * @param loopCount
     */
    public void play(String audioCode, Integer loopCount) {
        Assert.notNull(loopCount);
        Assert.checkBetween(loopCount.intValue(), 0, Integer.MAX_VALUE);
        Clip clip = getClip(audioCode);
        if (clip.isActive()) {
            return;
        }
        stop(audioCode);
        if (loopCount <= 1) {
            clip.start();
            return;
        }
        clip.loop(--loopCount);
    }

    /**
     * 停止音乐(立刻，马上)
     * @param audioCode
     */
    public void stop(String audioCode) {
        Clip clip = getClip(audioCode);
        /*if (!clip.isActive()) {
            return;
        }*/
        clip.stop();
        clip.flush();
        clip.setFramePosition(0);
    }

    /**
     * 调整音量（值为0.0表示最小音量，1.0表示最大音量）
     * @param audioCode
     * @param volume
     */
    public void setVolume(String audioCode, float volume) {
        Assert.checkBetween(volume, 0.0f, 1.0f);
        Clip clip = getClip(audioCode);
        // 计算音量值
        volume = (float) (Math.log(volume) / Math.log(10.0) * 20.0);
        FloatControl volumeControl = (FloatControl) clip.getControl(FloatControl.Type.MASTER_GAIN);
        volumeControl.setValue(volume);
    }


    public Clip getClip(String audioCode) {
        Assert.notBlank(audioCode);
        Clip clip = clipMap.get(audioCode);
        if (clip == null) {
            throw new RuntimeException("Uninitialized sound code.");
        }
        return clip;
    }
}

初始化安装 ​

程序通过判断application.properties的spring.profiles.active属性是否被赋值来区分是否完成初始化安装，即是否生成application-{程序模式}.properties环境的配置信息。

creatProperties函数中，通过TemplateUtil.createEngine模板引擎（默认使用了FreeMarker[1]）进行环境配置文件构建，如果文件已经存在即跳过生成逻辑，防止原文件被覆盖。

updateProp函数用于更新application.properties配置文件的spring.profiles.active属性，其中FileUtil.copy(path, copyPath, true);逻辑用于备份原文件。

java
/**
 * 初始化安装服务
 *
 * @author yuanzp
 * @date 2023/12/13 14:14
 * @digression 西宁曹家堡机场三期扩建工程信息弱电工程
 */
@Slf4j
@Service
public class InstallService {

    public static final String DEFAULT_APPLICATION_NAME = "application.properties";
    public static final String NEW_APPLICATION_NAME = "application-%s.properties";
    public static final String TEMPLATE_APPLICATION_NAME = "application-template.ftl";


    @Value("${spring.profiles.active}")
    private String profile;

    /**
     * 是否初始化
     * @return
     */
    public Boolean isInit() {
        return StrUtil.isNotBlank(profile);
    }
    
    /**
     * 构建生成Properties
     * @param propertiesReqDTO
     */
    public void creatProperties(PropertiesReqDTO propertiesReqDTO) {
        String profile = propertiesReqDTO.getProfile();
        String newApplicationName = String.format(NEW_APPLICATION_NAME, profile);
        String path = getClassPath(newApplicationName);
        boolean exist = FileUtil.exist(path);
        if (!exist) {
            //初始化安装
            TemplateEngine engine = TemplateUtil.createEngine(new TemplateConfig("", TemplateConfig.ResourceMode.CLASSPATH));
            Template template = engine.getTemplate(TEMPLATE_APPLICATION_NAME);
            template.render(BeanUtil.beanToMap(propertiesReqDTO), FileUtil.newFile(path));
            updateProp(profile);
        } else {
            log.warn("{}文件已经存在", newApplicationName);
        }
    }

    /**
     * 更新application.properties配置文件的<i>spring.profiles.active</i>属性
     * @param profile
     */
    public void updateProp(String profile) {
        String path = getClassPath(DEFAULT_APPLICATION_NAME);
        String copyPath = getClassPath(String.format(NEW_APPLICATION_NAME, DateUtil.format(DateUtil.date(), DatePattern.PURE_DATETIME_PATTERN)));
        FileUtil.copy(path, copyPath, true);
        Props props = PropsUtil.get(DEFAULT_APPLICATION_NAME);
        props.setProperty("spring.profiles.active", profile);
        props.store(path);

    }

    private String getClassPath(String fileName) {
        String path = ClassUtils.getDefaultClassLoader().getResource(StrUtil.EMPTY).getPath() + fileName;
        path = URLUtil.decode(path);
        return path;
    }
}

部署测试 ​

最终独立软件的成果物是“代码编译自测”章节打包出来的压缩包文件，解压缩工控机软件程序zip压缩包。

进入应用程序的安装目录（\pigm-cp\install），双击install.bat进行程序初始化安装（如果弹出验证框，点击允许，下文中执行bat文件出现类似提示同理），打印提示“successfully”即表示安装成功，并且启动成功（听语音提示判断程序是否完全启动）。

  • install.bat：安装服务（默认安装完成会自启动服务）
  • uninstall.bat：卸载服务（默认卸载前会自动停止服务）
  • start.bat：启动服务
  • restart.bat：重新启动服务
  • stop.bat：停止服务

浏览器访问工控机程序http://{服务IP}:8888/pigm-cp/，触发安装向导按钮进行初始化配置。

根据实际的外设信息配置相关信息，点击上一步，下一步可以切换不同的外设配置，最后保存配置并且重新启动工控机程序。

通过安装向导页面配置的信息会写入\pigm-cp\config中，如果需要重新安装程序，可以备份如下配置文件，即可跳过安装向导步骤。

  • application.properties：主配置文件
  • application-{时间戳}.properties：主配置文件备份（如果想要重新触发安装向导页面，即用该文件覆盖application.properties，并且删除application-{程序模式}.properties文件）
  • application-{程序模式}.properties：当前环境的配置信息，即安装向导页面的那些配置，也可以手动修改

进入\pigm-cp\logs目录，查看服务启动日志（顺便可以根据打印检查配置的程序服务IP是否正确）。

程序重启完成后，刷新浏览器页面，能够正常展示说明页面（显示外设测试按钮），即表示程序安装，启动完成。
