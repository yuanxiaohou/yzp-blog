一种基于sac和存储实现文件（图像、录像）转存的技术方案 ​

背景简介 ​

治超平台目前车辆称重数据超限和未超限是同一种模型数据（普通过车模型），即数据类型是一样，通过字段区分是否超限，因此图片存储依赖于一个管道，只能存储到同一个资源池里。录像则依赖于录像计划配置，通过监控点关联资源池的方式实现，同样无法实现精细化存储。此时无论是图片还是录像的时效性问题，都取决于存储设备所配置资源池的覆盖策略来决定。

但是用户在实际项目上，需要针对超限违法的数据，图片、录像需要存储的周期远大于未超限，这样需要平台支持将超限和未超限的图片分开存储。之前的做法一直都是以容量换取时长（推动用户增加存储去做图片长周期存储），这样一来存储容量无法最大化和最优化利用，存在一定的空间冗余浪费。很多项目上，项目规模较小，经费预算有限，用户不愿意去增加成本。

因此在此次治超OIM2.3产品中，实现将立案及结案后的图片、录像进行转存，以达到更小的存储空间来实现更精确的文件时效性存储控制。

优化方案的思考 ​

（1） 违法图片转存，平台支持将违法过车图片转存一份，转存后更新图片路径，可以针对已经审核过的违法数据做图片转存，减少无效数据（如无车牌、审核不通过数据等）的图片转存，提高存储空间的利用率。

（2） 从数据模型上区分开超限数据和未超限数据，将超限数据定义成违章数据，不同模型的数据，平台构架已支持配置不同存储池，但是要区分模型涉及 iac[1]、xfusion[2]、治超组件等众多组件的修改，可行性较低。

（3） 数据模型分开，但是只是 iac、设备层面分开，通过数据转换组件将iac 图片存储后的违章数据转换成普通过车再发一份到消息管道，后续应用组件不需修改。该方案需要 iac 简单定制，数据转换有现成组件可配置支持，无需修改组件代码。

以上 3 种方案，方案 2 经过了解沟通没有可行性。项目上可以采用方案 3，减少定制工作量。但是相对最优的方式还是方案1，组件自行进行转存，具备更好的灵活性和扩展性。

既然决定组件自行实现这些功能，那么接下来分析下具体的应用场景以及实现方案吧！

转存场景分析 ​

对于图片的转存，直接通过上传到存储设备即可实现。

但是通过录像计划录制的视频，却没有办法直接转存。而中心存储的统一管理通过sac[3]组件完成，所以切入点自然而然会想到从sac入手，通过hico上查阅了相关资料，大概整理出涉及的sac接口有：获取录像计划，录像锁定，手动补录。

录像转存场景 ​

方案设计 ​

总体数据流程设计 ​

异步任务&分段式缓存设计 ​

暴露给应用层的是一个创建任务接口createBacklogTask,内部进行分段式异步任务执行后续转存逻辑，而异步任务则通过Redis的ZSET有序队列行存储。任务过程中流转的缓存数据则通过对象进行存储。

任务默认为定时执行，也可以手动触发执行。

转存策略模式设计 ​

定义通用的转存策略接口CaseDumpStrategy进行转存，PicArchiveStrategy、VideoArchiveStrategy分别实现了图片和录像的转存逻辑，之后通过CaseDumpStrategyContext统一对外提供通过枚举获取相应转存实现的逻辑。

进度重试设计 ​

因为补录的接口为异步任务接口，所以对于进度的查询无法做到一次即成功，所以考虑采用重试，而重试的实现基于Spring-Retry，具体原理及使用可参考：Spring-Retry重试实现原理。

设备兼容性设计 ​

设备兼容性主要是通过sac和HikCStor[4]实现，应用端不做特殊处理。

📌备注

关于HikCStor接口定义及调用方式本文不做赘述，感兴趣的可自行查看wiki“海康存储/云存储协议（HikCStor）工具库”或者下载jar包查看。

数据备份设计 ​

转存场景中的图片转存和设备短视频转存两种场景中，因为会更新数据库中原始的结构化数据，所以为了防止误更新或者原数据丢失，需要对原始数据进行一次备份操作。

独立日志打印设计 ​

因为转存相对是一个比较独立的操作任务，为了方便问题的排查定位和统计相应任务执行状态，对转存逻辑中的日志单独打印到一个日志文件中。

方案实现 ​

分段式缓存实现 ​

采用这种设计的原因是本身转存中的“补录”就是异步任务类型的，所以就考虑将我们转存的业务也做成分阶段的缓存队列实现方式。

这样一来可以有效的将过程进行解耦分段，针对每个环节也能够更加方便有效的做一些优化，假如把整个过程笼统的写到一个方法内，也就很难单独对某个环节做改进和优化。

java
/**
 * 案件转存（过车图片+录像视频）
 * 拟采用异步延时队列（ZSET）方式实现，采用三段队列设计，核心任务处理器函数详见：<br/>
 * {@link #handler(TaskStatusEnum, TaskStatusEnum, Function<TaskCacheDTO, Boolean>)},<br/>
 * 三段逻辑设计如下：
 * <ol>
 *     <li>待办任务队列：backlog，处理函数详见：{@link #handleBacklogTask()}</li>
 *     <li>执行中任务队列：executory，处理函数详见：{@link #handleExecutoryTask()} ()}</li>
 *     <li>已完成任务队列：finished，处理函数详见：{@link #handleFinishedTask()} ()}</li>
 * </ol>
 * 目前已实现图片转存策略{@link PicArchiveStrategy}，视频转存策略{@link VideoArchiveStrategy}。
 * @author yuanzp
 * @date 2022/4/25 11:28
 * @digression done ibetter than perfect.
 */
@Slf4j
@Service
public class CaseDumpAlgo {
    @Autowired
    private TaskCacheRedisService taskCacheRedisService;

    @Autowired
    private TaskVehicleCacheRedisService taskVehicleCacheRedisService;

    @Autowired
    private CaseDumpStrategyContext caseDumpStrategyContext;

    /**
     * 创建待办任务
     */
    public void createBacklogTask(String passId, TaskTypeEnum taskTypeEnum, TaskResourceTypeEnum taskResourceTypeEnum) {
        CaseDumpStrategy<TaskCacheDTO> caseDumpStrategy = getCaseDumpStrategy(taskResourceTypeEnum);
        Boolean storageSwitch = caseDumpStrategy.storageSwitch();
        if (!storageSwitch) {
            log.debug("未打开转存开关，跳过创建任务逻辑！");
            return;
        }
        Date createTime = DateUtil.date();
        TaskCacheDTO taskCacheDTO = TaskCacheDTO.builder()
                .passId(passId).taskStatusEnum(TaskStatusEnum.BACKLOG).createTime(createTime)
                .taskTypeEnum(taskTypeEnum).taskResourceTypeEnum(taskResourceTypeEnum).build();
        taskCacheRedisService.add(TaskStatusEnum.BACKLOG, taskCacheDTO, createTime.getTime());
        taskVehicleCacheRedisService.set(taskCacheDTO);
        log.info("创建案件转存任务：{}", taskCacheDTO);
    }
    /*************************************以下任务从Redis中抓取数据处理*************************************/
    /**
     * 处理待办任务
     */
    public void handleBacklogTask() {
        handler(TaskStatusEnum.BACKLOG, TaskStatusEnum.EXECUTORY,
                (backlogTask) -> getCaseDumpStrategy(backlogTask.getTaskResourceTypeEnum()).archive(backlogTask));
    }

    /**
     * 处理执行中任务
     */
    public void handleExecutoryTask() {
        handler(TaskStatusEnum.EXECUTORY, TaskStatusEnum.FINISHED,
                (backlogTask) -> getCaseDumpStrategy(backlogTask.getTaskResourceTypeEnum()).progress(backlogTask));
    }

    /**
     * 处理已完成任务
     */
    public void handleFinishedTask() {
        handler(TaskStatusEnum.FINISHED, null,
                (backlogTask) -> getCaseDumpStrategy(backlogTask.getTaskResourceTypeEnum()).finished(backlogTask));
    }

    /**
     * 任务处理器
     * @param currentTask
     * @param nextTask
     * @param caseDumpStrategy
     */
    private void handler(TaskStatusEnum currentTask, TaskStatusEnum nextTask, Function<TaskCacheDTO, Boolean> caseDumpStrategy) {
        Set<String> keyList = taskCacheRedisService.range(currentTask, 0, -1);
        if (CollUtil.isEmpty(keyList)) {
            return;
        }
        keyList.forEach(key -> {
            TaskCacheDTO backlogTask = taskVehicleCacheRedisService.get(key);
            Boolean flag = false;
            try {
                flag = caseDumpStrategy.apply(backlogTask);
            } catch (Exception e) {
                log.error("任务节点执行失败。", e);
            }
            log.info("{},【{}】节点任务执行{}：{}", key, currentTask, flag, backlogTask);
            //当获取进度还在执行中或节点任务执行失败，不做任何数据移动。
            if (!flag) {
                return;
            }
            if (ObjectUtil.isNotNull(nextTask)) {
                backlogTask.setTaskStatusEnum(nextTask);
                taskCacheRedisService.add(nextTask, key, DateUtil.date().getTime());
                taskVehicleCacheRedisService.set(backlogTask);
            } else {
                taskVehicleCacheRedisService.delete(key);
            }
            taskCacheRedisService.removeValues(currentTask, key);
        });
    }

    private CaseDumpStrategy<TaskCacheDTO> getCaseDumpStrategy(TaskResourceTypeEnum taskResourceTypeEnum) {
        CaseDumpStrategy<TaskCacheDTO> caseDumpStrategy = caseDumpStrategyContext.getCaseDumpStrategy(taskResourceTypeEnum);
        return caseDumpStrategy;
    }
}

转存策略实现 ​

可以发现本次转存的场景分为两大类别，为了统一整体的转存过程，及定义统一的转存策略接口进行流程约束。

/**
 * 案件转存策略接口
 *
 * @author yuanzp
 * @date 2022/4/25 15:16
 */
public interface CaseDumpStrategy<T> {
    /**
     * 转存开关
     * @return
     */
    Boolean storageSwitch();

    /**
     * 文件转存
     * @return
     */
    Boolean archive(T t);

    /**
     * 存储进度
     * @return
     */
    default Boolean progress(T t) {
        return true;
    }

    /**
     * 完成
     * @return
     */
    Boolean finished(T t);
}

PicArchiveStrategy图片转存策略 ​

java
/**
 * 图片转存策略
 *
 * @author yuanzp
 * @date 2022/4/25 15:22
 * @digression done is better than perfect.
 */
@Slf4j
@Component
public class PicArchiveStrategy implements CaseDumpStrategy<TaskCacheDTO> {
    /**
     * 待转存图片属性字段名
     */
    private static final String[] PIC_FIELD = {"facePicUrl", "platePicUrl", "targetSubUrl", "vicePilotUrl",
            "vehiclePicUrl1", "vehiclePicUrl2", "vehiclePicUrl3", "vehiclePicUrl4", "vehiclePicUrl5", "vehiclePicUrl6"};

    @Autowired
    private HikcstorService hikcstorService;

    @Autowired
    private PicStorageConfigCacheRedisService picStorageConfigCacheRedisService;

    @Autowired
    private OverrunRecordMapper overrunRecordMapper;

    @Autowired
    private VehiclePassPicMapper vehiclePassPicMapper;

    @Autowired
    private BaseArchiveStrategy baseArchiveStrategy;

    @Override
    public Boolean storageSwitch() {
        Boolean picStorageSwitch = picStorageConfigCacheRedisService.picStorageSwitch();
        if (!picStorageSwitch) {
            log.warn("未打开案件图片转存开关。");
        }
        return picStorageSwitch;
    }

    @Override
    public Boolean archive(TaskCacheDTO taskCacheDTO) {
        TaskTypeEnum taskTypeEnum = taskCacheDTO.getTaskTypeEnum();
        StorageInfoDTO storageConfig = picStorageConfigCacheRedisService.picStorageConfig(taskTypeEnum);

        if (ObjectUtil.isNull(storageConfig)) {
            log.error("图片转存设备及储存池配置为空!");
            return false;
        }
        String passId = taskCacheDTO.getPassId();
        OverrunRecordEntity overrunRecordEntity = overrunRecordMapper.findByPassId(passId);
        if (ObjectUtil.isNull(overrunRecordEntity)) {
            log.warn("{}，overrunRecordMapper过车未查询到，退出图片转存!", passId);
            return false;
        }
        //存储通道规则：卡口编号_方向_车道
        String serialId = StrUtil.concat(true,
                overrunRecordEntity.getCrossingIndexCode(), StrUtil.UNDERLINE,
                overrunRecordEntity.getDirectionIndexCode(), StrUtil.UNDERLINE, overrunRecordEntity.getLaneNo());
        VehiclePassPicEntity vehiclePassPicEntity = vehiclePassPicMapper.findByPassId(passId);
        if (ObjectUtil.isNull(overrunRecordEntity)) {
            log.warn("{}，vehiclePassPicMapper过车图片记录未查询到，退出图片转存!", passId);
            return false;
        }
        Map<String, String> pictureMap = CollUtil.newHashMap();
        Map<String, String> beforePictureMap = CollUtil.newHashMap();
        for (String fieldName : PIC_FIELD) {
            Object fieldValue = ReflectUtil.getFieldValue(vehiclePassPicEntity, fieldName);
            String picUrl = Optional.ofNullable(fieldValue)
                    .filter(ObjectUtil::isNotEmpty).filter(s -> fieldValue instanceof String)
                    .map(String::valueOf).filter(PicArchiveStrategy::isHttpUrl).orElse(StrUtil.EMPTY);
            if (StrUtil.isBlank(picUrl)) {
                continue;
            }
            beforePictureMap.put(fieldName, picUrl);
            try {
                String uploadPicture = hikcstorService.uploadPicture(storageConfig.getDeviceID(), storageConfig.getPoolID(), serialId, picUrl);
                pictureMap.put(fieldName, uploadPicture);
            } catch (IOException e) {
                log.error("图片转存失败!", e);
                return false;
            }
        }
        taskCacheDTO.setPictureMap(pictureMap);
        taskCacheDTO.setBeforePictureMap(beforePictureMap);
        baseArchiveStrategy.insertCaseDumpBilling(taskCacheDTO);
        return true;
    }

    @Override
    public Boolean finished(TaskCacheDTO taskCacheDTO) {
        baseArchiveStrategy.updatePicUrl(taskCacheDTO);
        baseArchiveStrategy.updateCaseDumpBilling(taskCacheDTO);
        log.info("案件图片转存结束！\n{}", JSONUtil.toJsonStr(taskCacheDTO));
        return true;
    }

    private static Boolean isHttpUrl(String picUrl) {
        return Optional.ofNullable(picUrl).map(url -> url.toLowerCase().startsWith("http://")).orElse(false);
    }
}

VideoArchiveStrategy录像转存策略 ​

java
/**
 * 视频转存策略<br/>
 * 转存大致逻辑如下：
 * <ol>
 * <li>判断转存开关；</li>
 * <li>读取"录像存储方式"配置：<br/>
 *      1）"前端上传{@link VideoStorageModeEnum#DEVICE_UPLOAD}"方式直接下载短视频之后上传存储；<br/>
 *      2）"平台锁定{@link VideoStorageModeEnum#PLATFORM_LOCK}"方式对于中心存储的录像进行锁定；<br/>
 *      3）"平台补录{@link VideoStorageModeEnum#PLATFORM_PATCH}"方式对于设备存储进行补录；</li>
 * <li>更新过车表以及xfuison</li>
 * </ol>
 * @author yuanzp
 * @date 2022/4/25 15:22
 * @digression done is better than perfect.
 */
@Slf4j
@Component
public class VideoArchiveStrategy implements CaseDumpStrategy<TaskCacheDTO> {
    @Autowired
    private VideoStorageConfigCacheRedisService videoStorageConfigCacheRedisService;
    @Autowired
    private BaseArchiveStrategy baseArchiveStrategy;
    @Autowired
    private VideoArchiveService videoArchiveService;

    @Override
    public Boolean storageSwitch() {
        Boolean videoStorageSwitch = videoStorageConfigCacheRedisService.videoStorageSwitch();
        if (!videoStorageSwitch) {
            log.warn("未打开回放视频转存开关。");
        }
        return videoStorageSwitch;
    }

    @Override
    public Boolean archive(TaskCacheDTO taskCacheDTO) {
        TaskTypeEnum taskTypeEnum = taskCacheDTO.getTaskTypeEnum();
        StorageInfoDTO storageConfig = videoStorageConfigCacheRedisService.videoStorageConfig(taskTypeEnum);
        if (ObjectUtil.isNull(storageConfig)) {
            log.error("录像转存设备及储存池配置为空!");
            return false;
        }
        VideoStorageModeEnum videoStorageMode = videoStorageConfigCacheRedisService.videoStorageMode();
        if (ObjectUtil.isNull(videoStorageMode)) {
            log.error("录像转存方式无效！");
            return false;
        }
        switch (videoStorageMode) {
            case DEVICE_UPLOAD:
                return videoArchiveService.deviceUpload(storageConfig, taskCacheDTO);
            case PLATFORM_LOCK:
            case PLATFORM_PATCH:
                Boolean flag = videoArchiveService.platformHandleParambuilder(taskCacheDTO);
                if (ObjectUtil.isNotNull(flag)) {
                    return flag;
                }
                return VideoStorageModeEnum.PLATFORM_LOCK.equals(videoStorageMode)
                        ? videoArchiveService.platformLock(storageConfig, taskCacheDTO)
                        : videoArchiveService.platformPatch(storageConfig, taskCacheDTO);
            default:
                log.warn("无效的录像转存模式。");
                return true;
        }
    }

    @Override
    public Boolean progress(TaskCacheDTO taskCacheDTO) {
        String taskId = taskCacheDTO.getTaskId();
        if (StrUtil.isNotBlank(taskId)) {
            return videoArchiveService.platformPatchProgress(taskCacheDTO);
        }
        return true;
    }

    @Override
    public Boolean finished(TaskCacheDTO taskCacheDTO) {
        baseArchiveStrategy.updateVideoUrl(taskCacheDTO);
        baseArchiveStrategy.updateCaseDumpBilling(taskCacheDTO);
        log.info("案件录像转存结束！\n{}", JSONUtil.toJsonStr(taskCacheDTO));
        return true;
    }
}

📌备注

录像转存中的锁定和补录接口均通过Feign客户端的方式调用sac组件，具体如何自定义Feign客户端实现接口调用可以参考之前的wiki分享“通过Feign对接ISAPI协议接口的总结分享”。

java
/**
 * 视频转存服务
 *
 * @author yuanzp
 * @date 2022/5/20 8:39
 * @digression done is better than perfect.
 */
@Slf4j
@Component
@EnableRetry
public class VideoArchiveService {
    @Autowired
    private VideoStorageConfigCacheRedisService videoStorageConfigCacheRedisService;
    @Autowired
    private BaseArchiveStrategy baseArchiveStrategy;
    @Autowired
    private HikcstorService hikcstorService;
    @Autowired
    private RecordPlansRestServiceClient recordPlansRestServiceClient;
    @Autowired
    private RecordPlansCacheService recordPlansCacheService;
    @Autowired
    private LaneCacheRedisService laneCacheRedisService;
    /**
     * 案件审核录像锁定年限（默认2年）
     */
    @Value("${opunishment.lock-year.register:2}")
    private int lockYearForRegister;
    /**
     * 案件结案录像锁定年限（默认5年）
     */
    @Value("${opunishment.lock-year.case-finished:5}")
    private int lockYearForCaseFinished;

    /**
     * 前端上传逻辑
     * @param storageConfig
     * @param taskCacheDTO
     * @return
     */
    public Boolean deviceUpload(StorageInfoDTO storageConfig, TaskCacheDTO taskCacheDTO) {
        String passId = taskCacheDTO.getPassId();
        OverrunRecordEntity overrunRecordEntity = baseArchiveStrategy.findByPassId(passId);
        String vehicleVideoUrl = overrunRecordEntity.getVehicleVideoUrl();
        if (StrUtil.isBlank(vehicleVideoUrl)) {
            log.warn("{},前端上传的短视频URL为空，转存取消！", passId);
            return true;
        }
        taskCacheDTO.setBeforeVehicleVideoUrl(vehicleVideoUrl);
        File tempFile = FileUtil.file(passId.concat(StrUtil.DOT).concat("video"));
        hikcstorService.downloadFile(vehicleVideoUrl, tempFile);
        String newVehicleVideoUrl;
        try {
            newVehicleVideoUrl = hikcstorService.uploadObject(storageConfig.getDeviceID(), storageConfig.getPoolID(), tempFile);
        } catch (IOException e) {
            log.error("视频转存失败！", e);
            return false;
        } finally {
            FileUtil.del(tempFile);
        }
        taskCacheDTO.setVehicleVideoUrl(newVehicleVideoUrl);
        baseArchiveStrategy.insertCaseDumpBilling(taskCacheDTO);
        return true;
    }

    /**
     * 中心存储，采用锁定的方式
     * @param storageConfig
     * @param taskCacheDTO
     * @return
     */
    public Boolean platformLock(StorageInfoDTO storageConfig, TaskCacheDTO taskCacheDTO) {
        TaskTypeEnum taskTypeEnum = taskCacheDTO.getTaskTypeEnum();
        Integer lockYear = TaskTypeEnum.REGISTER.equals(taskTypeEnum) ? lockYearForRegister : lockYearForCaseFinished;
        Integer lockTime = NumberUtil.isGreater(BigDecimal.valueOf(lockYear), BigDecimal.ZERO)
                && NumberUtil.isLess(BigDecimal.valueOf(lockYear), BigDecimal.valueOf(50))
                ? NumberUtil.mul(lockYear, 365, 24, 60, 60).intValue()
                : SacConstants.MAX_LOCK_TIME;
        LockRecordReqDTO lockRecordReqDTO = LockRecordReqDTO.builder()
                .cameraIndexCode(taskCacheDTO.getCameraIndexCode()).lockTime(lockTime)
                .startTime(taskCacheDTO.getStartTime()).endTime(taskCacheDTO.getEndTime()).build();
        BaseResult baseResult = recordPlansRestServiceClient.lockRecord(lockRecordReqDTO);
        boolean isSuccess = Optional.ofNullable(baseResult).map(BaseResult::isSuccess).orElse(false);
        return isSuccess;
    }

    /**
     * 设备存储，采用补录的方式
     * @param storageConfig
     * @param taskCacheDTO
     * @return
     */
    public Boolean platformPatch(StorageInfoDTO storageConfig, TaskCacheDTO taskCacheDTO) {
        String poolID = storageConfig.getPoolID();
        ManualPatchReqDTO manualPatchReqDTO = ManualPatchReqDTO.builder()
                .cameraIndexCode(taskCacheDTO.getCameraIndexCode()).poolID(poolID)
                .startTime(taskCacheDTO.getStartTime()).endTime(taskCacheDTO.getEndTime()).build();
        BaseResult<ManualPatchRespDTO> manualPatchRespDTOBaseResult = recordPlansRestServiceClient.manualPatch(manualPatchReqDTO);
        String taskId = Optional.ofNullable(manualPatchRespDTOBaseResult).map(BaseResult::getData).map(ManualPatchRespDTO::getTaskID).orElse(StrUtil.EMPTY);
        taskCacheDTO.setTaskId(taskId);
        return StrUtil.isNotBlank(taskId);
    }

    /**
     * 补录进度及结果
     * @param taskCacheDTO
     * @return
     */
    @Retryable(include = {ExecutoryRetryException.class}, backoff = @Backoff(delay = 1000L, multiplier = 2))
    public Boolean platformPatchProgress(TaskCacheDTO taskCacheDTO) {
        BaseResult<PatchProgressRespDTO> baseResult = recordPlansRestServiceClient.patchProgress(taskCacheDTO.getTaskId());
        TaskStatusEnum taskStatusEnum = Optional.ofNullable(baseResult).map(BaseResult::getData).map(PatchProgressRespDTO::getTaskStatus).orElse(TaskStatusEnum.NONE);
        //未接入或者执行中状态，进行任务等待
        if (TaskStatusEnum.NONE.equals(taskStatusEnum) || TaskStatusEnum.EXECUTORY.equals(taskStatusEnum)) {
            //抛出执行中异常进行重试
            throw new ExecutoryRetryException("任务正在执行中。。。");
        }
        log.info("过车[{}]任务补录结果：{}({}).", taskCacheDTO.getPassId(), taskStatusEnum, taskStatusEnum.getCode());
        return true;
    }

    /**
     * 平台锁定和补录通用参数的构建
     * @param taskCacheDTO
     * @return true代表转存终止，false表示搁置，null表示继续操作
     */
    public Boolean platformHandleParambuilder(TaskCacheDTO taskCacheDTO) {
        String passId = taskCacheDTO.getPassId();
        //平台截取逻辑
        if (recordPlansCacheService.isEmpty()) {
            log.warn("平台未配置任何录像计划，录像转存取消！");
            return true;
        }
        OverrunRecordEntity overrunRecordEntity = baseArchiveStrategy.findByPassId(passId);
        //根据卡口编号，车道，方向获取监控点编号
        String cameraIndexCode = laneCacheRedisService.getRelatedCameraIndexCode(overrunRecordEntity.getCrossingIndexCode(),
                overrunRecordEntity.getDirectionIndexCode(), overrunRecordEntity.getLaneNo());
        if (StrUtil.isBlank(cameraIndexCode)) {
            log.warn("{},卡口车道未关联监控点，录像转存取消！", passId);
            return false;
        }
        Date passTime = DateUtil.date(overrunRecordEntity.getPassTime());
        Date startTime = DateUtil.offsetSecond(passTime, -videoStorageConfigCacheRedisService.videoSecondsBeforeIntercept());
        Date endTime = DateUtil.offsetSecond(passTime, videoStorageConfigCacheRedisService.videoSecondsAfterIntercept());

        RecordTypeEnum recordTypeEnum = Optional.ofNullable(recordPlansCacheService)
                .map(service -> {
                    //根据优先级，先获取是否有中心存储，再者获取设备存储
                    RecordPlansRespDTO recordPlansRespDTO = service.get(cameraIndexCode, RecordTypeEnum.CENTRAL);
                    recordPlansRespDTO = ObjectUtil.isNull(recordPlansRespDTO)
                            ? service.get(cameraIndexCode, RecordTypeEnum.DEVICE)
                            : recordPlansRespDTO;
                    return recordPlansRespDTO;
                }).map(RecordPlansRespDTO::getType).orElse(null);
        if (ObjectUtil.isNull(recordTypeEnum)) {
            log.warn("监控点:{}，未配置录像计划，录像转存取消！", cameraIndexCode);
            return true;
        }
        taskCacheDTO.setPassTime(passTime);
        taskCacheDTO.setCameraIndexCode(cameraIndexCode);
        taskCacheDTO.setStartTime(startTime);
        taskCacheDTO.setEndTime(endTime);
        return null;
    }
}

public void insertCaseDumpBilling(TaskCacheDTO taskCacheDTO) {
	CaseDumpBillingEntity caseDumpBillingEntity = CaseDumpBillingEntity.builder()
			.passId(taskCacheDTO.getPassId())
			.taskType(taskCacheDTO.getTaskTypeEnum().getCode())
			.resourceType(taskCacheDTO.getTaskResourceTypeEnum().getCode())
			.archiveContent(StrUtil.concat(true,
					Optional.ofNullable(taskCacheDTO.getBeforePictureMap())
							.map(JSONObject::toJSONString).orElse(null),
					taskCacheDTO.getBeforeVehicleVideoUrl()))
			.afterArchiveContent(StrUtil.concat(true,
					Optional.ofNullable(taskCacheDTO.getPictureMap())
							.map(JSONObject::toJSONString).orElse(null),
					taskCacheDTO.getVehicleVideoUrl()))
			.createTime(DateUtil.date())
			.build();
	caseDumpBillingMapper.insert(caseDumpBillingEntity);
	taskCacheDTO.setCdbId(caseDumpBillingEntity.getCdbId());
}

public void updateCaseDumpBilling(TaskCacheDTO taskCacheDTO) {
	Integer cdbId = taskCacheDTO.getCdbId();
	if (ObjectUtil.isNull(cdbId) || NumberUtil.isLessOrEqual(BigDecimal.valueOf(cdbId), BigDecimal.ZERO)) {
		return;
	}
	CaseDumpBillingEntity caseDumpBillingEntity = CaseDumpBillingEntity.builder()
			.cdbId(cdbId).finishTime(DateUtil.date()).build();
	caseDumpBillingMapper.updateByPrimaryKeySelective(caseDumpBillingEntity);
}

转存结果验证 ​

首先需要了解转存的设计核心是任务型，而创建任务的方式即触发案件复审通过或者案件结案操作。

任务创建之后会流转到一个待办任务队列中，之后通过“系统任务管理”模块中的“案件转存任务”任务触发后续的转存流程，默认为一小时执行一次，为了测试的便捷性，可以手动执行。

**无论是图片还是录像的转存验证，在验证功能性阶段，为了更有针对性，可以考虑挑选一条过车（记录其passId），观察其转存前后的变化来判断转存是否成功。**而对于程序的强壮性以及性能问题，可根据测试自有手段进行验证。

图片转存验证 ​

验证图片是否转存成功验证方式比较方便，即查看过车图片地址是否被更新成新地址，以及新的图片地址是否可以正常访问即可。

触发转存前后，查看MongoDB中的过车表BAYONET_VEHICLEPASS中的图片字段，以及转存记录表case_dump_billing记录，以此判断转存是否成功。

更加直观的方式是在“站点数据查询”模块查看转存前后图片是否可以正常访问。而关于图片存储时效性问题，由存储池的覆盖策略决定。

录像转存验证 ​

锁定 ​

录像锁定是针对于中心存储而言的，前提是监控点配置了相应的中心存储录像计划，以及产生了相应的录像片段。

对于锁定是否成功的判断依据，目前我已知的方式是查看sac录像回放接口返回参数中录像片段的**lockType（1:未锁定 2:已锁定 3:未知）**属性值进行分辨。

为了方便测试人员验证，在“站点数据查询”过车详细的“视频回放”接口中进行返回。所以验证锁定成功与否，可以在转存前后，分别F12查看/video/playback.do请求返回进行判断录像是否已锁定。

补录 ​

录像补录是针对于设备存储转存到中心存储的场景，前提是监控点配置了对应的设备存储录像计划和空模板的中心存储录像计划。

一旦触发补录成功，即中心存储也会有相应时间的录像片段。因此补录的验证方式即在“视频巡查”模块中的录像回放中，比较转存前后的录像片段：相应过车时间点转存前，中心录像片段为空，补录成功后中心存储也会产生一个录像片段。

还有一种方式是参考锁定章节的回放接口参数，查看**recordStyle（0-中心存储 1-设备存储）**属性进行分辨。

短视频 ​

短视频转存的验证方式和图片转存基本一致，属性字段不同而已，短视频URL复用BAYONET_VEHICLEPASS中devDescInfo字段。

短视频上传数据的模拟方式是在线模拟器中配置“设备描述”为一个可访问的视频地址（比如可以在tomcat的webapp中放一个MP4文件）。
